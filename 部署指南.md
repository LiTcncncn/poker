## 二、腾讯云部署方式

### 方式一：使用腾讯云 COS（对象存储）+ CDN（推荐）

#### 1. 准备工作
- 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
- 确保已开通 COS（对象存储）服务
- 建议同时开通 CDN（内容分发网络）加速

#### 2. 创建存储桶
1. 进入 [COS 控制台](https://console.cloud.tencent.com/cos)
2. 点击"创建存储桶"
3. 配置信息：
   - **名称**：自定义（如：poker-game）
   - **所属地域**：选择离用户最近的地域
   - **访问权限**：选择"公有读私有写"（静态网站需要公开访问）
   - **存储类型**：标准存储
4. 点击"确定"创建

#### 3. 配置静态网站托管
1. 进入创建的存储桶
2. 点击左侧"基础配置" → "静态网站"
3. 开启"静态网站"功能
4. 配置：
   - **索引文档**：`index.html`
   - **错误文档**：`index.html`（SPA应用需要）
5. 保存配置

#### 4. 上传文件
1. 进入存储桶的"文件列表"
2. 点击"上传文件"
3. 选择 `dist/` 目录下的所有文件：
   - `index.html`
   - `assets/` 目录（包含所有文件）
   - `pokers/` 目录（包含所有文件）
4. 上传方式：
   - **方式A（推荐）**：使用 COS 控制台上传
   - **方式B**：使用 [COS 命令行工具](https://cloud.tencent.com/document/product/436/15349)
   - **方式C**：使用 [COS Browser](https://cloud.tencent.com/document/product/436/11366)

#### 5. 获取访问地址
1. 在存储桶的"静态网站"配置中，找到"访问节点"
2. 访问地址格式：`https://<bucket-name>-<appid>.cos.<region>.myqcloud.com`
3. 或者使用自定义域名（需要配置域名解析）

#### 6. 配置 CDN 加速（可选但推荐）
1. 进入 [CDN 控制台](https://console.cloud.tencent.com/cdn)
2. 点击"添加域名"
3. 配置：
   - **加速域名**：填写你的域名（如：game.example.com）
   - **源站类型**：COS源
   - **源站地址**：选择刚才创建的存储桶
4. 配置完成后，等待域名审核通过
5. 在域名解析中添加 CNAME 记录，指向 CDN 提供的地址

### 方式二：使用腾讯云轻量应用服务器

#### 1. 准备工作
- 购买腾讯云轻量应用服务器（Linux系统）
- 确保服务器已安装 Nginx

#### 2. 上传文件到服务器
```bash
# 使用 scp 上传（在本地执行）
scp -r dist/* root@<服务器IP>:/var/www/html/

# 或者使用 FTP/SFTP 工具上传
```

#### 3. 配置 Nginx
编辑 Nginx 配置文件（通常在 `/etc/nginx/sites-available/default` 或 `/etc/nginx/nginx.conf`）：

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 替换为你的域名
    
    root /var/www/html;
    index index.html;
    
    # SPA路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

#### 4. 重启 Nginx
```bash
sudo nginx -t  # 测试配置
sudo systemctl restart nginx  # 重启服务
```

#### 5. 配置 HTTPS（推荐）
1. 使用腾讯云 SSL 证书服务申请免费证书
2. 在 Nginx 配置中添加 SSL 配置
3. 配置 HTTP 到 HTTPS 的重定向

## 四、常见问题

### 1. 页面空白
- **原因**：路径配置问题
- **解决**：确保 Nginx 配置了 SPA 路由支持（`try_files $uri $uri/ /index.html;`）

### 2. 资源404错误
- **原因**：文件路径不正确
- **解决**：检查 `dist/` 目录结构，确保所有文件都已上传

### 3. localStorage 不工作
- **原因**：浏览器安全策略
- **解决**：确保使用 HTTPS 访问（某些浏览器要求）

### 4. 移动端显示异常
- **原因**：viewport 配置问题
- **解决**：检查 `index.html` 中的 viewport meta 标签

## 五、性能优化建议

1. **启用 CDN 加速**：减少访问延迟
2. **配置缓存策略**：静态资源设置长期缓存
3. **启用 Gzip 压缩**：减少传输大小（COS/CDN 通常自动启用）
4. **使用 HTTPS**：提升安全性和 SEO
5. **配置域名**：使用自定义域名提升品牌形象
