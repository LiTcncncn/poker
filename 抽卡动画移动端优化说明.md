# 抽卡动画移动端优化说明

## 🎯 优化目标

**确保单抽和十连抽动画在移动端显示正常比例的卡牌**

## 📊 问题分析

### 优化前的问题

#### 1️⃣ **卡牌尺寸固定**
```tsx
// Card 组件的 showDetails 模式（抽卡时使用）
showDetails ? 'w-28 h-40 sm:w-32 sm:h-44' : '...'
```

**问题**：
- 移动端：`w-28 h-40` (112×160px)
- 配合 `scale-150` 缩放：112 × 1.5 = **168px**
- 在小屏幕（如 375px）上占比过大：168/375 = **44.8%**
- ❌ 卡牌太大，超出屏幕范围
- ❌ 比例与主界面不一致

#### 2️⃣ **十连抽布局固定**
```tsx
// Draw10Animation 固定 5 列布局
<div className="grid grid-cols-5 gap-4">
```

**问题**：
- 5 列 × 112px + 4 个间距 × 16px = 560 + 64 = **624px**
- 在 1179px 屏幕勉强能放下
- 在小屏幕（< 600px）上会挤在一起
- ❌ 间距太大，浪费空间

#### 3️⃣ **标题和文字尺寸固定**
- 标题：`text-3xl` / `text-4xl`
- 图标：`w-8 h-8`
- ❌ 移动端显得过大，占用太多空间

## ✅ 解决方案

### 1️⃣ Card 组件 - 响应式 showDetails 尺寸

#### 修改前
```tsx
showDetails ? 'w-28 h-40 sm:w-32 sm:h-44' : '...'
```

#### 修改后
```tsx
showDetails ? 'w-20 h-28 sm:w-32 sm:h-44' : '...'
```

**效果**：
- **移动端**：`w-20 h-28` (80×112px) - 与主界面卡牌一致！
- **PC 端**：`sm:w-32 sm:h-44` (128×176px) - 保持原大小
- ✅ 移动端卡牌更小，比例正常
- ✅ 与主界面保持一致的视觉感受

---

### 2️⃣ DrawAnimation (单抽) - 多项优化

#### 优化 A：标题响应式

**修改前**：
```tsx
className="text-3xl font-bold mb-12"
<Sparkles className="w-8 h-8" />
```

**修改后**：
```tsx
className="text-2xl sm:text-3xl font-bold mb-8 sm:mb-12 px-4"
<Sparkles className="w-6 h-6 sm:w-8 sm:h-8" />
```

**效果**：
- 移动端：文字 text-2xl (24px)，图标 w-6 (24px)
- PC 端：文字 text-3xl (30px)，图标 w-8 (32px)
- ✅ 移动端更紧凑，不占用过多空间

#### 优化 B：卡牌缩放响应式

**修改前**：
```tsx
stage === 'show' && "scale-150"
```

**修改后**：
```tsx
stage === 'show' && "scale-125 sm:scale-150"
```

**效果**：
- **移动端**：80px × 1.25 = **100px** ✅ 合理
- **PC 端**：128px × 1.5 = **192px** ✅ 醒目
- ✅ 移动端不会过大
- ✅ PC 端保持震撼效果

**实际卡牌大小对比**：

| 设备 | 基础尺寸 | 缩放 | 最终尺寸 | 屏占比 |
|------|---------|------|---------|--------|
| **小手机 (375px)** | 80px | ×1.25 | 100px | 26.7% ✅ |
| **大手机 (1179px)** | 80px | ×1.25 | 100px | 8.5% ✅ |
| **PC (1920px)** | 128px | ×1.5 | 192px | 10% ✅ |

---

### 3️⃣ Draw10Animation (十连抽) - 多项优化

#### 优化 A：标题响应式

**修改前**：
```tsx
<h2 className="text-4xl font-bold">十连抽结果</h2>
<Sparkles className="w-8 h-8" />
<div className="flex gap-6 text-lg">
```

**修改后**：
```tsx
<h2 className="text-2xl sm:text-4xl font-bold">十连抽结果</h2>
<Sparkles className="w-6 h-6 sm:w-8 sm:h-8" />
<div className="flex gap-3 sm:gap-6 text-sm sm:text-lg">
```

**效果**：
- 移动端：更小的文字和图标
- PC 端：保持原样
- ✅ 为卡牌网格留出更多空间

#### 优化 B：网格布局优化

**修改前**：
```tsx
<div className="grid grid-cols-5 gap-4 mb-8">
```

**修改后**：
```tsx
<div className="grid grid-cols-5 gap-2 sm:gap-4 mb-8 max-w-full overflow-x-auto px-2">
```

**效果**：
- 移动端：`gap-2` (8px) - 更紧凑
- PC 端：`gap-4` (16px) - 保持宽松
- `max-w-full` - 防止超出屏幕
- `overflow-x-auto` - 超出时可横向滚动
- `px-2` - 左右留白，防止贴边

**布局计算**：

| 设备 | 卡牌尺寸 | 列数 | 间距 | 总宽度 | 屏幕宽度 | 状态 |
|------|---------|------|------|--------|---------|------|
| **小手机** | 80px | 5 | 8px×4 | 432px | 375px | 可滚动 ✅ |
| **大手机** | 80px | 5 | 8px×4 | 432px | 1179px | 完美显示 ✅ |
| **PC** | 128px | 5 | 16px×4 | 704px | 1920px | 完美显示 ✅ |

## 💡 响应式策略

### 关键点 1：统一基础尺寸

**移动端**（< 1320px）:
```tsx
// 主界面
w-[18vw] max-w-20  // 最大 80px

// 抽卡动画（showDetails）
w-20               // 80px

// ✅ 完全一致！
```

**PC 端**（≥ 1320px）:
```tsx
// 主界面
sm:w-24            // 96px

// 抽卡动画（showDetails）
sm:w-32            // 128px (更大，更醒目)

// ✅ 合理差异化
```

### 关键点 2：灵活缩放

```tsx
// 单抽动画
scale-125 sm:scale-150

// 移动端: 80px × 1.25 = 100px (不会太大)
// PC 端: 128px × 1.5 = 192px (震撼效果)
```

### 关键点 3：紧凑布局

```tsx
// 移动端优先的间距
gap-2 sm:gap-4     // 8px vs 16px
mb-4 sm:mb-8       // 16px vs 32px

```
