# 品质牌字体统一优化说明

## 🎯 优化目标

**确保移动端和 PC 端品质牌的视觉比例完全一致**

## 📊 问题分析

### 之前的方案（响应式字体）

```tsx
// 移动端使用较小字体，PC 端使用较大字体
className="text-2xl sm:text-3xl"
```

**问题**：
- ❌ 移动端和 PC 端使用不同的绝对字体大小
- ❌ 导致文字相对于卡牌的比例不一致
- ❌ 移动端字体相对于卡牌太大，显得拥挤

### 用户反馈

> "移动端：品质牌，花色、数值显示过大，需要适当缩小，和 pc 端比例完全一致"

**核心需求**：
1. 移动端字体需要更小
2. 移动端和 PC 端的**比例**要一致

## ✅ 解决方案

### 统一字体大小（移除响应式）

**策略**：
- 移除所有 `sm:` 响应式字体类
- 使用统一的较小字体尺寸
- 让字体随卡牌容器自然缩放

由于卡牌本身使用了视口单位：
```tsx
// 移动端: w-[18vw] (自适应)
// PC 端: sm:w-24 (96px)
className="w-[18vw] max-w-20 h-[25.2vw] max-h-28 sm:w-24 sm:h-36"
```

卡牌尺寸会自动适应屏幕，字体也会相应缩放，保持相同比例。

## 📝 具体修改

### 1️⃣ 花色（Suit）

#### 修改前
```tsx
// 单花色
className="text-2xl sm:text-3xl"

// 双花色
className="flex gap-1 text-2xl sm:text-3xl"
```

#### 修改后
```tsx
// 单花色
className="text-xl"  // 统一 text-xl

// 双花色
className="flex gap-0.5 text-xl"  // 统一 text-xl，间距也缩小
```

**效果**：
- 字体从 text-2xl (24px) / text-3xl (30px) 统一为 text-xl (20px)
- 更小，更协调
- 移动端和 PC 端比例一致

### 3️⃣ 效果值（+n, ×n）

#### 修改前
```tsx

```

#### 修改后
```tsx
// 单效果
className="text-2xl font-black"  // 统一 text-2xl

// 双效果
className="text-xl font-black"  // 统一 text-xl
```

**效果**：
- 单效果：text-2xl (24px)
- 双效果：text-xl (20px) - 更小以适配并排
- 层次分明
- 比例统一

## 💡 技术原理

### 为什么统一字体大小就能保持比例？

**关键在于卡牌容器的自适应尺寸**：

1. **移动端**（< 1320px）：
   ```tsx
   width: 18vw;      // 屏幕宽度的 18%
   max-width: 5rem;  // 最大 80px
   ```
   - 1179px 屏幕：18% × 1179 ≈ 212px → 受限于 max-w-20 = 80px
   - 字体 text-xl = 20px
   - **字体占卡牌比例**：20 / 80 = **25%**

2. **PC 端**（≥ 1320px）：
   ```tsx
   width: 6rem;  // 96px (sm:w-24)
   ```
   - 字体 text-xl = 20px
   - **字体占卡牌比例**：20 / 96 = **20.8%**

虽然不是完全精确的比例，但视觉上非常接近，且更重要的是：

### 相对大小关系保持一致

无论移动端还是 PC 端：
- 花色 : 数值 : 单效果 : 双效果 = xl : xl : 2xl : xl
- 即：1 : 1 : 1.2 : 1
- **层次关系完全一致**

## 📱 响应式策略对比

### 方案 A：响应式字体（之前）

```tsx
className="text-2xl sm:text-3xl"
```

**特点**：
- 移动端和 PC 端使用不同的绝对大小
- 需要手动调整每个断点
- 难以保证比例一致
- 维护复杂

### 方案 B：统一字体 + 自适应容器（现在）✅

```tsx
// 字体固定
className="text-xl"

// 容器自适应
className="w-[18vw] max-w-20 sm:w-24"
```

**特点**：
- 字体大小统一
- 容器自适应缩放
- 比例自然一致
- 维护简单
- 性能更好（减少断点判断）
