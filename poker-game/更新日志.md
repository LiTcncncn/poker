### 积分规则优化 ⚡
修改了积分计算逻辑，现在**只计算参与牌型的牌**的数值和倍数：
- **一对**：只计算对子的2张（其他3张的数值和倍数都不计）
- **两对**：只计算两对的4张（踢脚牌不参与）
- **三条**：只计算三条的3张（其他2张不参与）
- **四条**：只计算四条的4张（踢脚牌不参与）
- **高牌**：只计算最高的1张
- **同花/顺子/葫芦/同花顺/皇家同花顺**：计算全部5张

**重要**：倍数牌效果也只对参与牌型的牌生效！
- 例：一对K + 倍数牌5(不参与) = (10+10) × 2 = 40分
- 倍数牌5虽然有×2效果，但因为没参与牌型，所以效果不生效

**影响**：
1. 牌型之间的积分差距更加合理
2. 高级牌型的价值更加突出
3. 倍数牌的位置变得更加重要

## 📋 技术细节

### 项目结构
```
src/
├── components/
│   ├── Card.tsx           # 卡牌组件（3D翻转）
│   ├── DeckView.tsx       # 牌池界面
│   ├── StatsView.tsx      # 统计界面
│   ├── SkillTree.tsx      # 技能树界面
│   └── HandAnimation.tsx  # 牌型动画
├── store/
│   └── gameStore.ts       # Zustand 状态管理
├── types/
│   └── poker.ts           # TypeScript 类型定义
├── utils/
│   └── pokerLogic.ts      # 牌型判定算法
├── App.tsx                # 主应用组件
├── index.css              # 全局样式和动画
└── main.tsx               # 入口文件
```

### 技术栈
- **前端框架**：React 18 + TypeScript
- **状态管理**：Zustand
- **样式方案**：Tailwind CSS
- **图标库**：Lucide React
- **构建工具**：Vite

### 核心算法
- **牌型判定**：回溯算法处理特殊牌组合
- **积分计算**：牌面数值 × (基础倍率 + 品质倍率)
- **体力恢复**：时间戳计算，5秒/次

## 🎯 使用说明

### 启动开发服务器
```bash
cd poker-game
npm install
npm run dev
```

### 游戏操作
1. **翻牌**：点击"翻牌"按钮或直接点击卡牌区域
2. **自动模式**：点击"自动"按钮启用挂机模式
3. **抽卡**：消耗100积分抽取绿色/蓝色品质卡牌
4. **查看牌池**：点击右上角的网格图标
5. **查看统计**：点击右上角的柱状图图标
6. **查看技能**：点击右上角的奖杯图标
