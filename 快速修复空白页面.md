# 快速修复：腾讯云访问空白页面

## ⚠️ 最常见问题：上传方式错误

### 问题原因
你上传了 `dist` **文件夹本身**，而不是 `dist` **里面的内容**。

### ✅ 正确做法

#### 方法1：使用 COS 控制台（推荐）

1. **删除错误上传**：
   - 进入存储桶 → 文件列表
   - 如果看到 `dist/` 文件夹，删除它

2. **正确上传**：
   - 在本地打开 `dist` 文件夹
   - 选择 `dist` 文件夹**里面的所有内容**：
     - ✅ `index.html`
     - ✅ `assets` 文件夹（整个文件夹）
     - ✅ `pokers` 文件夹（整个文件夹）
   - 上传到存储桶的**根目录**（不是 dist 文件夹内）

3. **最终文件结构应该是**：
   ```
   存储桶根目录/
   ├── index.html          ← 直接在根目录
   ├── assets/
   │   ├── index-0c7QtiHj.css
   │   └── index-Bt7rPuNA.js
   └── pokers/
       └── (所有图片文件)
   ```

#### 方法2：使用命令行（如果你熟悉）

```bash
# 进入 dist 目录
cd dist

# 上传 dist 里面的内容到存储桶根目录
# 注意：是上传 . (当前目录内容)，不是 dist 文件夹
coscmd upload -rs . /
```

## 🔧 其他必须检查的配置

### 1. 静态网站托管配置

进入：存储桶 → 基础配置 → 静态网站

- ✅ **静态网站**：已开启
- ✅ **索引文档**：`index.html`
- ✅ **错误文档**：`index.html`（重要！SPA应用必须）

### 2. 使用正确的访问地址

- ✅ 使用静态网站提供的**访问节点**
- ❌ 不要使用存储桶的默认访问地址

**如何找到正确地址**：
存储桶 → 基础配置 → 静态网站 → 查看"访问节点"

地址格式：`https://<bucket-name>-<appid>.cos.<region>.myqcloud.com`

## 🧪 验证步骤

1. **检查文件结构**：
   - 在 COS 控制台，存储桶根目录应该直接看到 `index.html`
   - 不应该看到 `dist` 文件夹

2. **检查静态网站配置**：
   - 索引文档：`index.html`
   - 错误文档：`index.html`

3. **测试访问**：
   - 使用静态网站访问节点
   - 打开浏览器开发者工具（F12）
   - 查看 Console 是否有错误
   - 查看 Network 是否所有资源都加载成功

## 📸 正确 vs 错误对比

### ❌ 错误的结构
```
存储桶根目录/
  └── dist/              ← 错误！不应该有 dist 文件夹
      ├── index.html
      ├── assets/
      └── pokers/
```
访问时路径变成：`/dist/index.html`，资源路径 `/assets/...` 找不到！

### ✅ 正确的结构
```
存储桶根目录/
  ├── index.html         ← 直接在根目录
  ├── assets/
  └── pokers/
```
访问时路径：`/index.html`，资源路径 `/assets/...` 正确！

   - Network 标签：看哪些资源 404
3. **提供信息**：
   - 浏览器控制台的错误截图
   - 存储桶文件结构截图
