# 宝塔面板部署指南

## 问题：访问 dist/index.html 显示空白

### 问题原因
`index.html` 中的资源路径是绝对路径 `/assets/...`，当你访问 `dist/index.html` 时：
- 页面路径：`/dist/index.html`
- 资源路径：`/assets/...`（指向网站根目录，不是 `/dist/assets/`）
- 结果：资源找不到，页面空白

---

## 解决方案（三种方法）

### 方案一：将 dist 内容移动到网站根目录（推荐⭐）

**步骤：**

1. **登录宝塔面板**
   - 进入"文件"管理

2. **找到你的网站根目录**
   - 通常是：`/www/wwwroot/你的域名/`
   - 或者：`/www/wwwroot/你的网站目录/`

3. **删除或备份现有文件**
   - 如果根目录有其他文件，先备份或删除

4. **移动 dist 文件夹内容到根目录**
   - 进入 `dist` 文件夹
   - 选择 `dist` 文件夹**里面的所有内容**：
     - `index.html`
     - `assets` 文件夹
     - `pokers` 文件夹
   - 剪切或复制这些文件
   - 返回到网站根目录
   - 粘贴到根目录

5. **最终文件结构应该是**：
   ```
   /www/wwwroot/你的域名/
   ├── index.html          ← 直接在根目录
   ├── assets/
   │   ├── index-0c7QtiHj.css
   │   └── index-Bt7rPuNA.js
   └── pokers/
       └── (所有图片文件)
   ```

6. **访问网站**
   - 直接访问你的域名：`http://你的域名/`
   - 或：`http://你的域名/index.html`

---

### 方案二：配置 Nginx，将网站根目录指向 dist 文件夹

**步骤：**

1. **登录宝塔面板**
   - 进入"网站" → 找到你的网站 → 点击"设置"

2. **修改网站根目录**
   - 点击"网站目录"
   - 将"网站运行目录"改为：`/www/wwwroot/你的域名/dist`
   - 保存

3. **配置 Nginx（重要！）**
   - 点击"配置文件"
   - 找到 `location /` 部分，修改为：
   ```nginx
   location / {
       root /www/wwwroot/你的域名/dist;
       index index.html;
       try_files $uri $uri/ /index.html;
   }
   ```
   - 保存并重启 Nginx

4. **访问网站**
   - 直接访问：`http://你的域名/`
   - 会自动指向 `dist/index.html`

---

### 方案三：修改构建配置，重新构建（不推荐）

如果你坚持要访问 `dist/index.html`，需要修改构建配置：

1. **修改 vite.config.ts**：
   ```typescript
   import { defineConfig } from 'vite'
   import react from '@vitejs/plugin-react'
   import path from 'path'

   export default defineConfig({
     base: '/dist/',  // 添加这行
     plugins: [react()],
     resolve: {
       alias: {
         '@': path.resolve(__dirname, './src'),
       },
     },
   })
   ```

2. **重新构建**：
   ```bash
   npm run build
   ```

3. **上传新的 dist 文件夹**

4. **访问**：`http://你的域名/dist/index.html`

**注意**：这种方法不推荐，因为每次部署都需要重新构建。

## 推荐方案：方案一

**为什么推荐方案一？**
- ✅ 最简单，不需要修改配置
- ✅ 符合标准部署方式
- ✅ 访问路径更简洁（直接访问域名）
- ✅ 不需要重新构建

## 宝塔面板具体操作步骤（方案一）

### 1. 进入文件管理
- 登录宝塔面板
- 点击左侧"文件"

### 2. 找到网站目录
- 通常路径：`/www/wwwroot/你的域名/`
- 或者：`/www/wwwroot/你的网站目录/`

### 3. 进入 dist 文件夹
- 找到 `dist` 文件夹
- 双击进入

### 4. 选择所有内容
- 在 `dist` 文件夹内，选择：
  - `index.html` 文件
  - `assets` 文件夹
  - `pokers` 文件夹
- 点击"复制"或"剪切"

### 5. 返回到网站根目录
- 点击路径栏的上级目录，回到网站根目录

### 6. 粘贴文件
- 点击"粘贴"
- 确认文件已移动到根目录

### 7. 删除空的 dist 文件夹（可选）
- 如果 `dist` 文件夹已空，可以删除

### 8. 验证文件结构
- 确认网站根目录现在有：
  - `index.html`
  - `assets/` 文件夹
  - `pokers/` 文件夹

### 9. 访问网站
- 浏览器访问：`http://你的域名/`
- 应该能看到游戏界面

## 配置 Nginx（如果需要 SPA 路由支持）

如果你的应用有前端路由，需要配置 Nginx 支持：

1. **进入网站设置**
   - 宝塔面板 → 网站 → 你的网站 → 设置

2. **点击"配置文件"**

3. **找到 `location /` 部分，确保有**：
   ```nginx
   location / {
       try_files $uri $uri/ /index.html;
   }
   ```

4. **保存并重启 Nginx**

## 常见问题

### Q: 访问后还是空白？
**A:** 检查浏览器控制台（F12）：
- Console 标签：看是否有错误
- Network 标签：看哪些资源 404
- 确认资源路径是否正确

### Q: 图片不显示？
**A:** 检查：
- `pokers/` 文件夹是否在根目录
- 图片文件是否完整上传
- 文件权限是否正确（应该是 644）

### Q: 刷新页面 404？
**A:** 配置 Nginx 的 `try_files`：
```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```
