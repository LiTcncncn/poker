# 成就系统设计方案

## 1. 系统概述

成就系统是一个渐进式奖励机制，玩家通过达成各种牌型来解锁成就档位，获得钻石奖励。钻石是游戏中的新代币，显示在主界面顶部与金钱并排。

## 2. 核心功能

### 2.1 钻石代币
- **位置**：主界面顶部，与金钱（$）并排显示
- **图标**：使用 `Gem` 图标（lucide-react）
- **用途**：成就系统奖励代币（未来可扩展其他用途）

### 2.2 成就类型
共9种牌型成就，每种30档：

| 牌型 | 每档奖励 | 档位阈值（30档） |
|------|---------|----------------|
| 一对 | 50 钻 | 1, 10, 20, 30, 40, 50... |
| 两对 | 50 钻 | 1, 8, 16, 24, 32, 40, 48... |
| 三条 | 100 钻 | 1, 5, 10, 15, 20, 25... |
| 顺子 | 200 钻 | 1, 3, 6, 9, 12, 15, 18... |
| 同花 | 200 钻 | 1, 3, 6, 9, 12, 15, 18... |
| 葫芦 | 300 钻 | 1, 3, 6, 9, 12, 15, 18... |
| 四条 | 500 钻 | 1, 2, 4, 6, 8, 10... |
| 同花顺 | 1000 钻 | 1, 2, 3, 4, 5... |
| 皇家同花顺 | 2000 钻 | 1, 2, 3, 4, 5... |

### 2.3 成就机制
- **检测时机**：每次翻牌结算后，根据 `stats[handType].count` 检测成就进度
- **领取方式**：手动领取，每个档位独立领取
- **进度追踪**：记录每个牌型当前完成的最高档位和已领取的档位

## 3. 数据结构

### 3.1 成就配置
```typescript
interface AchievementConfig {
  handType: HandType;
  reward: number; // 每档奖励钻石数
  thresholds: number[]; // 30个档位的阈值数组
}
```

### 3.2 成就进度
```typescript
interface AchievementProgress {
  handType: HandType;
  currentTier: number; // 当前完成的最高档位 (0-29)
  claimedTiers: number[]; // 已领取的档位索引数组
}
```

### 3.3 游戏状态扩展
```typescript
// 在 GameState 中添加
diamonds: number; // 钻石数量
achievements: Record<HandType, AchievementProgress>; // 成就进度
```

## 4. UI 设计

### 4.1 主界面顶部
```
[$ 1000] [💎 500] [⚡ 20/20]
```
- 钻石显示格式：`[💎 500]`
- 与金钱并排，样式一致

### 4.2 按钮区域
```
[统计] [牌池] [合成] [技能] [成就] [重置]
```
- 成就按钮位置：技能按钮右侧
- 图标：`Award` 或 `Medal`（lucide-react）
- 提示点：有可领取成就时显示红点提示

### 4.3 成就界面
- **布局**：移动端优化，响应式设计
- **左侧**：牌型列表（9种牌型，可滚动）
- **右侧/下方**：选中牌型的详细进度
  - 当前进度：`X / Y`（当前次数 / 下一档要求）
  - 进度条：可视化进度
  - 档位列表：显示所有30档
    - 已完成未领取：高亮显示，可点击领取
    - 已完成已领取：灰色显示，已领取标记
    - 未完成：灰色显示，显示要求次数
  - 奖励显示：每档奖励钻石数

## 5. 交互流程

1. **翻牌结算** → 更新 `stats[handType].count`
2. **检测成就** → 检查是否有新完成的档位
3. **更新进度** → 更新 `achievements[handType].currentTier`
4. **显示提示** → 成就按钮显示红点提示
5. **打开界面** → 玩家点击成就按钮
6. **查看进度** → 选择牌型查看详细进度
7. **手动领取** → 点击可领取的档位，获得钻石奖励
8. **更新状态** → 将档位加入 `claimedTiers`，增加钻石数量

## 6. 技术实现

### 6.1 文件结构
- `src/utils/achievements.ts` - 成就配置和工具函数
- `src/components/AchievementView.tsx` - 成就界面组件
- `src/store/gameStore.ts` - 添加钻石和成就状态管理
- `src/App.tsx` - 添加钻石显示和成就按钮

### 6.2 关键函数
- `checkAchievements(handType: HandType, count: number)` - 检测成就进度
- `claimAchievement(handType: HandType, tier: number)` - 领取成就奖励
- `getClaimableTiers(handType: HandType)` - 获取可领取的档位列表
- `hasClaimableAchievements()` - 检查是否有可领取的成就

### 6.3 数据持久化
- 钻石数量需要持久化到 localStorage
- 成就进度需要持久化（已完成档位、已领取档位）
- 使用 zustand persist 中间件

## 7. 移动端优化

- 响应式布局：使用 Tailwind CSS 的 `sm:` 断点
- 触摸优化：按钮大小至少 44x44px
- 滚动优化：牌型列表和档位列表支持平滑滚动
- 字体大小：移动端适当缩小，保持可读性
- 间距优化：移动端减少间距，PC端增加间距
