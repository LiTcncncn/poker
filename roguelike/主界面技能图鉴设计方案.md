# 主界面「技能」图鉴设计方案

> **状态**：**已实现（2026-06-11，v0.3）**  
> **范围**：主界面 `RunEntry` 入口；全屏 `SkillCodexView` 展示 **附加边（4）+ 技能牌（80）**。  
> **约束**：手机竖屏、`max-w-[390px]`；不展示解锁顺序号、进度条、「第 N 局解锁」规则句。

---

## 1. 目标

| 状态 | 展示 |
|------|------|
| **已解锁** | 正面内容；技能点按 → `SkillPlayingCardDetailModal`；已解锁边点按 → 底栏说明弹层 |
| **未解锁** | 牌背 + **仅「？」**；**点击弹出居中弹层**展示解锁条件（普通通关「局名」） |

---

## 2. 已定决策（v0.3）

| # | 决策 |
|---|------|
| Q1 | 顶栏 **普通/困难 Tab 右侧**「技能」 |
| Q2 | 附加边：**边名 + 流光条**，非完整技能牌 |
| Q3 | 已解锁技能 → **`SkillPlayingCardDetailModal`**（无翻转） |
| Q4 / Q12 | 未解锁 **仅 ？**；**点击弹层**显示解锁条件（局中文名） |
| Q5 | 解锁进度 **仅** `profile.highestNormalRunCleared` |
| Q6 / Q14 | 引导未完成 **仍显示**入口；顺序 1 共 29 张绿牌 **默认正面** |
| Q7 | 附加边 **`grid-cols-4` 居中**（max-w 320px） |
| Q9 | 小节标题 **「附加边」「技能牌」** |
| Q11 | 已解锁边 → **底栏弹层**（边名 + 效果一句） |
| Q15 | 引导期 **`[ 普通 ]` + `[ 技能 ]` 同一行** |
| 排序 | 技能 **绿 → 蓝 → 紫**，同品质按 **`runSkillPool.json` 顺序** |
| Q13 / 7日轮回 | 仍占 **1 格**；图鉴局名 **「难上加难」**（第 4 局，**无新技能顺序解锁**）；**不使用第 51 局** |

---

## 3. 入口与页面

- `RunEntry` 顶行：`[ 普通 | 困难 ]` 或引导期 `[ 普通 ]` + 右端 **技能**。
- 进入 `SkillCodexView`；**← 返回** 回选局大厅（保留局号/难度）。

---

## 4. 解锁判定

### 4.1 附加边

| 边 | 通关局号 ≥ | 局名 |
|----|-----------|------|
| 银 `flash` | 4 | 难上加难 |
| 金 `gold` | 6 | 牌型陷阱 |
| 彩 `laser` | 8 | 十面埋伏 |
| 黑 `black` | 10 | 无小丑恶梦 |

### 4.2 技能

- `getUnlockedOrdersAfterNormalRun(highestNormalCleared)` + `SKILL_UNLOCK_ORDER_MAP`。
- **`seven_cycle_x4`（7 日轮回）**：主线 **永不解锁**；图鉴点击 **？** 后显示 **「难上加难」**（`SEVEN_CYCLE_CODEX_DISPLAY_RUN_NO = 4`），**不改** `skillUnlockOrders` 公式。

### 4.3 技能排序

绿 33 → 蓝 26 → 紫 21（`listSkillsForCodex()`）。

## 7. 验收要点

1. Tab 行右侧「技能」；引导期单行布局 OK。  
2. 附加边 4 列 + 标题；技能 5 列 × 80。  
3. 未解锁默认 **？**；点击弹出 **解锁条件**（普通通关「局名」）；牌背不直接显示局名。  
4. 7 日轮回未解锁，点后显示 **难上加难**（非 51 局）。  
5. 已解锁技能/边详情弹层正常；Profile 解锁与商店池一致。
