# 关后钻石结算板设计方案

> 面向策划 / UI / 程序。**仅手机竖屏**（内容区 `max-w-[390px]` 单列）。  
> **状态**：**已实现（2026-06-11，v0.3.1）**  
> **关联**：`roguelikeStore.ts`、`StageView.tsx`、`RewardModal.tsx`。

---

## 1. 目标与边界

### 1.1 要做什么

在 **单关胜利、且即将进入关后商店** 的流程中，**自动弹出**「钻石结算板」，解释本关 💎 来源：

| 条目 | 说明 |
|---|---|
| **基础（普通/精英/Boss）** | 通关基础 💎（含局级覆盖、词缀归零时为 0） |
| **剩余手数（N）** | 括号内为剩余手数；右侧为加成 💎（可为 0） |
| **剩余补牌（N）** | 括号内为剩余补牌次数；右侧为加成 💎（可为 0） |
| **钻石王老五** | 单独一行 💎5（持有才显示） |
| **钻石牌** | 本关超级钻牌累计；**仅当 > 0 时显示一行** |
| **本关合计** | 以上各项之和（加粗） |

玩家点 **「继续」** 后进入统一商店。

### 1.2 不展示结算板

| 场景 | 行为 |
|---|---|
| 关内失败 → `RunResult` | 否 |
| 主线 **最后一关** 胜利（无商店） | 否 |
| **IAA 复活** 过关（无胜利 💎） | **跳过**，直接进商店 |
| **引导第 1 关**（`tutorialRun` 且 stageIndex=0） | **否** — 保留现有「获得 💎N！」总述蒙层 |
| 主界面 / 自由挑战入口 | 否 |

### 1.3 已确认原则

- 明细数字与 **实际入账** 一致；本关内累计钻牌 💎（`StageState.diamondCardEarnedThisStage` 或等价字段）。
- 剩手/剩补牌加成为 **0 仍显示一行**，右侧 **💎0**。
- **钻石牌** 本关累计 **为 0 时不展示该行**（与王老五「无则省略」一致）。
- **不展示**「胜利奖励小计」「当前持有」及底部 footnote 说明行。
- 本版 **不做 EN / language.json**。
- 实现时同步修正：IAA 额外手数/Hold **不参与** 胜利钻石剩手/剩补牌加成。

## 2. 钻石公式（实现基准）

### 2.1 关后胜利奖励 — `calcStageDiamondReward`

```
若 shopBaseDiamondRewardZero → base=0，handBonus=0，holdBonus=0

base =
  stageBaseDiamondReward（局级覆盖，如第 14 局精英 = 1）
  否则 精英/Boss → 5，普通关 → 3

handsLeft = 有效剩余手数（剔除 IAA 额外手）
handBonus = handsLeft≥2 → +2；=1 → +1；否则 0

holdLeft = 有效剩余补牌（剔除 IAA 额外 Hold）
holdBonus = holdLeft≥1 → +1；否则 0

stageWinSubtotal = base + handBonus + holdBonus   // 仅用于算合计，UI 不单独展示
```

### 2.2 关后技能 — `applyStageWinEconomy`

- **钻石王老五**：💎5（单独一行）

### 2.3 关内钻牌 — 每手结算时入账

- 超级 **钻牌** 参与计分时按 `diamondBonus` 加进 `runDiamonds`。
- 本关累计 > 0 时在结算板显示 **「钻石牌」** 一行；为 0 则 **省略**。

### 2.4 本关合计

```
本关合计 totalGranted =
  base + handBonus + holdBonus + tycoonBonus + stageDiamondCardEarned
```

（`stageWinSubtotal` 与上式前三项等价，不在 UI 拆开展示小计行。）

## 3. 目标流程

```
最后一手 scoreHand → stage won
  → handState.phase = 'result'
  → 【自动】弹出 StageDiamondSettlement（遮住底栏）
  → 用户点「继续」
  → setRewardVisible(true) → RewardModal 商店
```

| 分支 | 结算板 |
|---|---|
| 无尽中间关 | **展示** |
| 引导第 1 关 | **不展示** → 蒙层点「下一关」直接进商店 |
| IAA 复活 | **跳过**，直接进商店 |
| 主线末关胜利 | 无结算板、无商店 |

## 4. 结算板文案与行结构

### 4.1 标题区

- `第 N 关 · 普通` / `第 N 关 · 精英` / `第 N 关 · BOSS` / `无尽第 N 关 · 精英`

### 4.2 明细行（固定顺序）

| 顺序 | 行 key | 左侧文案 | 右侧 |
|---:|---|---|---|
| 1 | `base` | `基础（普通）` / `基础（精英）` / `基础（Boss）` | 💎n |
| 2 | `hands` | `剩余手数（N）` — N 为剩余手数 | 💎n（可为 0） |
| 3 | `hold` | `剩余补牌（N）` — N 为剩余补牌次数 | 💎n（可为 0） |
| 4 | `tycoon` | `钻石王老五` | 💎5（**无技能则整行省略**） |
| 5 | `diamond_cards` | `钻石牌` | 💎n（**本关无钻牌收益则整行省略**） |
| — | `total` | **本关合计** | 💎n（加粗） |

**不展示**：胜利奖励小计、当前持有、词缀/局规则 footnote。

### 4.3 基础行括号内类型

| 关卡类型 | 左侧文案 |
|---|---|
| 普通关 | `基础（普通）` |
| 精英关 | `基础（精英）` |
| Boss 关 | `基础（Boss）` |

局级覆盖（如第 14 局精英 💎1）、词缀「钻石0奖励」等 **只影响右侧数值**，左侧仍按关类型显示；归零时 `💎0`。

### 4.4 主按钮

- 文案：**「继续」**
- 仅按钮可点；点击遮罩不关闭

## 5. UI 方案（手机）

- `fixed inset-0 z-[140]` 半透黑 + 居中卡片 `max-w-[390px]`
- result 底栏「下一关」隐藏，由结算板「继续」接管
- 数字 `tabular-nums`；明细行与 **本关合计** 右侧仅显示 `💎n`（**不加** `+` 前缀）；合计用 `text-rl-gold` 加粗
- 商店 `RewardModal` z-index 高于结算板

---

## 6. 数据接口

```ts
export interface StageDiamondBreakdown {
  stageIndex: number;
  stageLabel: string;              // 「第 3 关 · 精英」
  base: number;
  handBonus: number;
  holdBonus: number;
  handsLeft: number;
  holdLeft: number;
  stageKind: 'normal' | 'elite' | 'boss';
  tycoonBonus: number;
  stageDiamondCardEarned: number;
  totalGranted: number;
  skipBoard?: boolean;             // 引导第 1 关等
}
```

**累计钻牌**：`_commitScore` 每手 `handDiamond > 0` 时累加至当前关。  
**生成时机**：关胜利分支，与入账同事务。  
**存储**：`RLStore.pendingStageDiamondBreakdown`（非持久化）。

## 9. 验收要点

1. `result` 后自动弹板；按钮 **「继续」** 进商店。
2. 基础行：`基础（普通）` / `基础（精英）` / `基础（Boss）`。
3. 剩余手数/补牌：`剩余手数（N）`、`剩余补牌（N）`；💎0 行仍显示。
4. **无** 胜利奖励小计、当前持有、footnote。
5. 王老五、钻石牌均为 **有收益才显示行**；本关合计 = 已展示各行之和。
6. 词缀钻石 0：仍弹板，各行 💎0。
7. 引导第 1 关无结算板；IAA 复活跳过；末关/失败无板；无尽有板。
