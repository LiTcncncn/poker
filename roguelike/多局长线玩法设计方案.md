# Roguelike 多局长线玩法设计方案（草案）

> 本文只做规划，不实施代码。目标是把当前“单局 20 关 + 通关后无尽”的玩法，扩展成 50 局主线、普通/困难双模式、每局胜利后进入该局无尽冲榜的长期目标系统。

---

## 1. 现有系统理解

### 当前玩法结构

- `src/components/RunEntry.tsx` 是当前主入口：标题、玩法说明、“开始游戏”和“冲关排行”。
- `src/App.tsx` 按是否存在 active run 决定展示入口或 `StageView`，目前没有“第 n 局 / 普通困难 / 局外解锁”选择层。
- `src/types/run.ts` 中的 `RunState` 保存单局技能、牌型升级、超级牌、局内钻石、精英词缀、无尽状态；`StageState` 保存单关目标分、手数、Hold 数、禁 Joker、牌型降级等。
- `src/config/runStageTemplates.json` 是当前 20 关主线曲线：目标分从 `$200` 到 `$40000`，默认每关 6 手、6 Hold，第 3/6/9/12/15/18/20 关为精英或 Boss。
- `src/config/runEliteModifiers.json` 已有大量可复用限制：禁牌型、Hold -1/-2/-3、手数 -1、禁 Joker、目标分 x1.2/x1.4、牌型降级等。
- `src/engine/stageEngine.ts` 负责单关胜负：累计 `$` 达到 `getStageTargetGold(stage)` 胜利，手数用尽未达标失败。
- `src/engine/runEngine.ts` 负责新局、推进关卡、主线胜利后进入无尽；无尽目标分已经有动态递增公式。
- `src/engine/rewardEngine.ts` 负责关后统一 6 卡商店，含技能、牌型升级、超级牌；技能边框支持银/金/彩/黑，黑边可扩技能槽。
- `src/store/roguelikeStore.ts` 负责串联发牌、结算、钻石、商店、进入无尽。本局钻石开局 3，普通关基础 +3，精英/Boss 基础 +5，并按剩余手数/Hold 追加。

### 已有文档约束

- `已有技能列表.md` 是技能和效果的权威资料，共 80 条技能，并要求与 `runSkillPool.json` 双向同步。
- `界面改版-本关目标与结算区.md` 强调手机竖屏窄栏、不增加高度、结算数值来源要可解释。
- `技能牌牌面测试页方案.md` 可作为“外观解锁 / 技能牌皮肤预览”的后续基础。
- 工作区规则要求 Roguelike UI 只按手机竖屏设计，主内容保持约 `max-w-[390px]`，不做桌面断点布局。

### 当前适合扩展的方向

现有系统已经适合扩展“每局一个规则包”：

- 目标分：已有 `targetGold` 和 `targetGoldMultiplier`。
- 手数 / Hold：已有 `totalHands`、`holdTotal` 和对应词缀。
- Joker：已有 `banJokers`。
- 牌型等级：已有牌型升级与临时降级。
- 无尽：已有主线通关后继续挑战和本地排行雏形。

当前缺失的是“局外长线层”：

- `ProfileState`：保存第几局已通关、困难是否解锁、技能牌永久解锁、外观解锁、每局无尽最高纪录。
- `RunConfig`：按第 n 局、普通/困难、特殊牌堆生成不同 run。
- RNG seed：后续如果做全球排行校验、录像、每日挑战，需要可复现随机。

## 2. 长线玩法目标

1. **持续目标**：玩家始终知道下一局要挑战什么、能解锁什么、自己在哪一局卡住。
2. **低数值膨胀**：由于基础玩法没有跨局数值成长，目标分提升要温和，难度主要来自规则组合。
3. **构筑变化**：普通局也可以有额外限制；这些限制是每局的核心差异玩法，用来制造新鲜感，而不是只靠目标分递增。
4. **挑战与炫耀分层**：普通局推进主线，困难局和每局无尽提供挑战、外观、排行价值。
5. **不强迫困难**：普通局按顺序推进；同一局普通胜利后解锁该局困难，但困难胜利不作为后续普通局前置。

推荐核心循环：

1. 主界面选择第 n 局。
2. 默认在“普通”页签，展示本局名称、限制条件、奖励预览，不展示目标分具体值。
3. 点击“挑战”进入该局普通主线。
4. 普通胜利后解锁下一普通局、当前局困难、当前局无尽入口，并按后续技能解锁表发放奖励。
5. 胜利结算先弹出“继续无尽 / 结算返回”，不强制自动进入无尽；如果选择继续无尽，本次普通胜利刚解锁的技能包与界面边框应立即对当前局生效。
6. 困难胜利奖励暂不拍板，后续再议；困难通关与困难无尽成绩仍需要记录。

## 3. 普通 / 困难结构

### 普通模式

- 共 50 局，顺序挑战。
- 第 1 局为新手局，10 关。
- 第 2 到第 50 局均为 20 关。
- 普通局可以配置额外限制，这些限制是每局主题玩法的主体，目标是有差异但不过度重复。
- 普通局胜利后先弹出“继续无尽 / 结算返回”；选择继续时进入当前局规则下的无尽关卡，并实时使用本次通关刚解锁的技能池与界面边框。服务器端全球排行暂不实现，先保留本地最高纪录与未来字段。

### 困难模式

- 每一局的困难模式在该局普通胜利后解锁。
- 困难不阻塞后续普通局。
- 困难不是单纯提高目标分，而是在普通局规则之上视情况叠加第二层限制：
  - 目标分略高。
  - 去掉普通局的部分补偿。
  - 可增加一条轻到中等限制，但不是每局都必须叠加。
  - 精英/Boss 关使用更高难度词缀池。
- 困难奖励暂不拍板，后续再议；原则上不应阻塞普通主线推进。

## 4. 目标分设计

### 基础曲线

当前 20 关目标分曲线可作为第 2 局以后的基础：

`200, 400, 650, 900, 1300, 1800, 2500, 3200, 4100, 5400, 6600, 8000, 9600, 11800, 14100, 16700, 19900, 23500, 33000, 40000`

第 1 局是 10 关新手局，建议使用单独低压曲线：

`120, 250, 450, 700, 1000, 1450, 2100, 3000, 4200, 6000`

### 普通模式目标倍率

第 50 局普通不建议超过当前基准曲线的 1.35 倍，更多压力来自规则。

- 第 1 局：使用新手 10 关曲线。
- 第 2-10 局：`0.92x -> 1.02x`
- 第 11-20 局：`1.02x -> 1.10x`
- 第 21-35 局：`1.10x -> 1.20x`
- 第 36-50 局：`1.20x -> 1.32x`

### 困难模式目标倍率

困难目标分基于同局普通目标分再乘：

- 第 1-10 局：`1.10x`
- 第 11-25 局：`1.16x`
- 第 26-40 局：`1.22x`
- 第 41-50 局：`1.28x`

困难还会叠加规则限制，因此不建议再大幅拉高目标分。

### 特殊规则数值原则

特殊规则不默认用“目标分打折”来补偿。长线主线的基础逻辑是“规则变化 + 目标分逐步提升”共同形成难度，所以特殊规则应先被当作本局主题来评估，而不是自动降目标。

- `只有 ♥/♠`：这是偏增益的花色归并规则，不是减益。除 Joker 外仍使用 52 张牌，最终形成 26 张 ♥️ 与 26 张 ♠️；原 ♦️ 牌转为同点数 ♥️，原 ♣️ 牌转为同点数 ♠️，等于每个点数都有重复的 ♥️/♠️。所有牌型可能性会提高，尤其是同花，因此不做目标分折扣补偿。
- `A-10 局`：牌堆为 A、2-10，去掉 J/Q/K，人头牌技能失效，皇家同花顺不可成；按该局主题重新配置技能池/局内限制，不默认降低目标。
- `7-A 局`：牌堆为 7、8、9、10、J、Q、K、A，低牌减少，牌面分和高阶牌型变强；可承受略高目标或额外限制。
- `无 Joker`：作为明确减益时，优先通过 Joker 技能降权、避免重复禁 Joker、给非目标类小补偿来处理，不默认降低目标。
- `Hold -2` 或 `手数 -2`：除非同时给强非目标补偿，否则不建议出现在前 20 局普通。

## 5. 局规则组件库

### 用户预设规则

以下规则来自最初设定，优先级高，默认纳入长线主线设计。

#### 减益

- 增加目标分限制：`$ +5% / +10% / +15% / +20%`（具体档位可再调）。
- `Hold -1 / -2`
- `手数 -1 / -2`
- 无 Joker

#### 增益

- 减少目标分限制：`$ -5% / -8% / -12%`（作为独立增益规则保留，但不作为强减益的默认补偿手段）。
- `Hold +1 / +2`
- `手数 +1 / +2`
- 开局钻石 = 10
- 技能牌 +1
- 所有牌型 Lv.2 起步

#### 特殊规则

- 只有 `♥/♠`：花色归并，除 Joker 外仍使用 52 张牌，形成 26 张 ♥️ 与 26 张 ♠️；原 ♦️ 转为同点数 ♥️，原 ♣️ 转为同点数 ♠️，会出现重复的 ♥️/♠️ 牌。
- `A-10 局`：牌堆只保留 A、2-10，移除 J/Q/K。
- `7-A 局`：牌堆只保留 7、8、9、10、J、Q、K、A。

### 待确认补充规则

以下规则是我基于现有代码能力和长线玩法补充的候选项，需要你确认是否纳入。

#### 减益候选

- 禁某类牌型：禁高牌、禁一对、禁两对、禁顺子线、禁同花线、禁葫芦。
- 商店刷新费用 +3。
- 所有商店商品价格 +2。
- 通关基础钻石降为 💎1。
- 开局钻石为 0。
- 奖励压力：随机去掉 2 个商店商品，可表现为商店中随机 2 个商品变暗且不可购买。

### 局外规则与局内限制叠加原则

局外规则定义“这一局怎么玩”，局内限制定义“这一关临时怎么变”。两者不能无脑叠加，尤其是减益规则需要按限制轴做预算，避免出现“局外已经砍掉操作空间，局内再砍同一轴”的情况。

#### 限制轴分类

- 资源轴：手数、Hold、前后手不可补牌。
- 分数轴：目标分倍率。
- 牌堆轴：无 Joker、特殊花色、特殊点数范围。
- 牌型轴：禁高牌、禁一对、禁两对、禁顺子线、禁同花线、禁葫芦等。
- 经济轴：钻石收入、刷新费用、技能价格、商店候选数量。

#### 普通模式叠加上限

- 单关最终不低于 `4 手 / 3 Hold`。
- 单关最终目标倍率不超过 `1.35x`。
- 同一关最多 2 条强限制。
- 如果局外规则已经是强限制，局内限制只能选轻限制或非同轴限制。
- 前 20 局普通不建议出现 `手数 -2`，也不建议让最终 Hold 低于 3。

#### 困难模式叠加上限

- 单关最终不低于 `4 手 / 2 Hold`。
- 单关最终目标倍率不超过 `1.6x`。
- 同一关最多 3 条限制，但不能全部落在同一限制轴。
- 困难可以叠第二层限制，但不是每局必须叠。
- 如果局外已经有强资源压缩，困难优先叠牌型轴、牌堆轴或经济轴，不继续压手数/Hold。

#### 冲突与过滤规则

- 局外无 Joker 时，局内不再抽 `ban_jokers`。
- 局外无 Joker 时，Joker 技能在本局商店中应降权或暂时屏蔽，除非该局明确设计为“通关后为下一局准备”。
- 特殊花色规则生效时，局内不抽会禁掉核心花色的限制。例如只有 `♥/♠` 时，不抽禁 `♥` 或禁 `♠`。
- 黑牌局只保留 `♠/♣` 时，不抽禁 `♠` 或禁 `♣`；红牌局同理。
- `A-10 局` 不叠人头无效，因为 J/Q/K 已经不在牌堆里。
- `7-A 局` 谨慎叠禁顺子线，除非该局本身就是高牌、对子或三条构筑主题。
- 局外 `Hold -2` 时，局内不再抽 `Hold -2/-3` 或前后 2 手不可补牌。
- 局外 `手数 -1/-2` 时，局内不再抽手数 -1，除非该关有明确的非目标分补偿。
- 局外经济减益存在时，局内不再叠同轴经济减益，例如商店刷新费用 +3、所有商店商品价格 +2、通关基础钻石降为 💎1、随机去掉 2 个商店商品。
- 局外 `$ +15%` 以上时，精英/Boss 不再抽 `high_target_12` 或 `high_target_14`。

#### 补偿原则

当局外规则是强减益时，至少给一个轻补偿，避免玩家只感到被削弱。补偿优先使用非目标分手段，不把“目标分打折”作为默认平衡方式。

- `Hold +1` 或 `手数 +1`。
- 开局钻石 = 10。
- 技能槽 +1。
- 所有牌型 Lv.2 起步。
- 商店首刷免费。

#### 实现建议

后续实现时，不建议让精英/Boss 词缀完全随机。每个主线局应带 `ruleTags` 和 `banModifierTags`：

- `ruleTags` 描述本局主题，例如 `no_joker`、`low_hold`、`two_suits_hearts_spades`、`a_to_10`、`shop_pressure`。
- `banModifierTags` 描述本局禁止叠加的局内限制，例如 `ban_joker_modifiers`、`ban_same_suit_bans`、`ban_more_hold_loss`、`ban_high_target`。
- 生成精英/Boss 词缀前先过滤冲突词缀，再按难度池抽取。

## 7. 奖励与解锁设计

### 技能牌解锁

建议不要开局就把 80 张技能全部放入普通主线池，而是用局外解锁建立长期动力。技能解锁以你整理的 `1-27` 解锁顺序为准，作为“技能包顺序”融入 1-50 局普通局。

- 默认已解锁：解锁顺序 1 作为默认技能池，第 1 局和第 2 局默认可用。
- 普通胜利：按解锁顺序解锁；由于技能数量有限，不要求每一局都解锁技能。
- 普通胜利后若选择继续无尽，本次刚解锁的技能包会立即加入当前 run 的无尽商店候选池，而不是等到返回主界面后才生效。
- 技能附加边与技能池同样走局外进度门槛；本局商店只会 Roll 出 `allowedSkillEnhancements` 中已开放的边。继续无尽时，本次普通胜利刚解锁的边会立即加入当前 run 的无尽商店候选池。
- 关键 Boss 局：可优先安排更强主题奖励，例如技能、外观、牌桌、称号或终章展示内容。
- 困难胜利：奖励后续再议，暂不在本文拍板。

这样可以让技能作为普通主线推进奖励，同时给外观、称号、无尽纪录留下足够空间。

#### 技能解锁顺序与普通局映射（建议版）

采用“解锁顺序 1 默认已解锁；第 2 局普通胜利解锁顺序 2；第 3 局普通胜利解锁顺序 3；之后基本隔 1 局解锁一次；第 50 局普通胜利解锁顺序 27”的节奏。这样第 1、2 局有基础技能池可玩，第 2、3 局连续给新目标，中后期保持稳定长线期待，最后一局有压轴技能奖励。

## 9. 技术落点建议（后续实现前再确认）

### 新增 ProfileState

建议保存：

- `highestNormalRunCleared`
- `normalClears: Record<runNo, clearInfo>`
- `hardClears: Record<runNo, clearInfo>`
- `unlockedSkillIds`
- `unlockedCosmetics`
- `bestEndlessRecords: Record<modeKey, record>`
- `selectedCosmetics`
- `schemaVersion`

不需要新增局外货币；若未来有局外资源，也应优先只服务外观，不进入主线强度。

### 新增 RunConfig

建议每次开始挑战时生成：

- `runNo`
- `difficulty: normal | hard | freeplay`
- `stageCount`
- `targetMultiplier`
- `ruleSet`
- `rewardUnlocks`
- `allowedSkillEnhancements`
- `allowEndless`
- `rulesetVersion`
- `seed`

### 无尽阶段索引

- 每局进入无尽时，无尽关卡固定写入 `stages[runStageCount]`，而不是固定写入标准 20 关索引。
- 第 1 局新手局 `runStageCount = 10`，因此第一关无尽使用索引 10，并在界面显示为第 11 关。
- 第 2～50 局 `runStageCount = 20`，因此第一关无尽使用索引 20，并在界面显示为第 21 关。
- 关卡展示、发牌恢复、无尽推进、无尽记录统计都应读取 `runStageCount`，避免 10 关新手局继续无尽时找不到当前关。

### 新增 MainlineRunDef

建议把 50 局定义为数据：

- `id`
- `runNo`
- `title`
- `subtitle`
- `normalConfig`
- `hardConfig`
- `unlockRewards`
- `skillUnlockOrder`
- `endlessConfig`
- `displayTags`

这样后续主界面、开局、结算、无尽排行都读同一份配置。

## 10. 风险与注意点

- **目标分不能膨胀太快**：当前单局内强度已经来自技能/升级/超级牌，跨局不提供数值成长时，50 局目标上限应保守。
- **特殊牌堆会强烈影响概率**：只有两种花色、A-10、7-A、奇偶局都不是简单难/易，需要实际跑几轮手感后微调。
- **技能锁池会影响平衡**：如果早期锁掉太多强技能，20 关后段可能偏难；需要保证每个阶段至少有 2-3 条可用构筑路线。
- **困难奖励暂未拍板**：困难可以提供长期动力，但不应成为普通主线推进的必要条件。
- **无尽排行需要规则隔离**：每一局、普通/困难、规则版本都应是不同榜，否则成绩不可比。
- **文档同步**：如果后续改技能池、关卡模板、词缀或奖励权重，需要同步更新 `已有技能列表.md` 或对应玩法文档。

## 11. 已确认决策与待补资料

### 已确认

1. 技能解锁采用你整理的 `1-27` 顺序；解锁顺序 1 默认已解锁，供第 1、2 局使用；第 2 局普通胜利解锁顺序 2，之后融合到 1-50 局普通胜利奖励中。

5. 选择继续无尽时，本次普通通关刚解锁的技能包与界面边框实时生效。
