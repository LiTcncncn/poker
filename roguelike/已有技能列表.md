## 蓝色（blue）

| id | 名称 | 描述 |
|----|------|------|
| `heart_mult` | ♥️牌加倍 | 计分时每张 ♥ 额外 +3 倍 |
| `spade_mult` | ♠️牌加倍 | 计分时每张 ♠ 额外 +3 倍 |
| `club_mult` | ♣️牌加倍 | 计分时每张 ♣ 额外 +3 倍 |
| `diamond_mult` | ♦️牌加倍 | 计分时每张 ♦ 额外 +3 倍 |
| `face_mult` | 人头牌加倍 | 计分时每张人头牌（J/Q/K）额外 +3 倍 |
| `face_score` | 人头牌加注 | 计分时每张人头牌（J/Q/K）额外 +$30 |
| `odd_score` | 奇数牌加注 | 计分时每张奇数点数牌额外 +$30 |
| `even_score` | 偶数牌加注 | 计分时每张偶数点数牌额外 +$30 |
| `ace_combo` | A加注加倍率 | 计分时每张 A 额外 +4 倍且 +$30 |
| `joker_rate` | Joker几率+ | 提升 JOKER 的出现几率。 |
| `joker_double` | JOKER 成双 | 一手牌中可能出现两张 JOKER（计分与手牌上限逻辑不变）。 |
| `skill_count_mult` | 技能数加倍 | 每拥有 1 个技能，额外 +3 倍。 |
| `nine_accum` | 逢9累积 | 每次结算含 9，每张 9 累积 +$20（上限 $200），并加入本次得分 |
| `odd_straight_skill` | 奇数成顺 | 纯奇数牌（A/3/5/7/9）可成顺子；兼容同花顺。并且计分时额外 +10 倍 |
| `even_straight_skill` | 偶数成顺 | 纯偶数牌（2/4/6/8/10）可成顺子；兼容同花顺。并且计分时额外 +10 倍 |
| `remaining_hands_mult` | 剩手加倍 | 结算时获得本关剩余手数 x2 的加倍。 |
| `sellers_market` | 卖方市场 | **蓝色**。每次关卡胜利：每个已拥有技能的卖出回收价 +1💎（不改变商店购买价）；**每个技能 id 该项累计上限 +8💎**。 |
| `diamond_score_per_gem` | 钻石估值 | 每持有 1💎（计分取样时刻，不含本手待发钻），本手 +$1。 |
| `diamond_mult_per_gem` | 钻石加倍 | 每持有 **3💎**（同上取样，**向下取整**），本手 **+1** 倍（加性倍率，非独立乘区）；`perRunDiamondsCost: 3`。 |
| `super_broke` | 穷神附身 | 计分时若局内 💎≤3，独立乘区 ×1.5。 |
| `second_hand_joker_x2` | 次手Joker爆发 | 本关第 2 手结算：若 Joker 参与计分，结算 ×2 |
| `high_straight_x3` | 高阶顺子乘倍 | 10JQKA 顺子（四张成顺时 JQKA），结算 ×3。 |
| `refresh_shop_mult` | 刷新商店乘倍 | 每次使用 💎 刷新商店，乘倍+0.2，上限 x2。 |
| `fading_boost` | 后劲不足 | 装备后：第 1 手 +$100，第 2 手 +$95，第 3 手 +$90……每手 −$5，直到 +0 后该技能自动销毁。（口径：按“手”递减；仅加性 $） |
| `forward_rush` | 勇往直前 | 本关总手数 +2，同时本关补牌总量 −1。（口径：进入每关时生效；与词缀/精英关无限制交互待实现一致） |

## 类型参考（改 JSON 时用）

见 `src/types/skill.ts` 中 `SkillEffectType`、`RuleModType` 定义。

- `accumulate_score_saved_hands`：`value` 为每「剩余 1 手」增加的技能池数值，`accumulateCap` 为池上限；在本关因本手结算而**达标通关**时，按 `剩余手数 = 本关总手数 - 本手结算后的已用手数` 叠加入池，并将**整池**并入当次技能$（实现见 `applySavedHandsStageWinBonus` / `_commitScore`）。
- `accumulate_multiplier_no_draw`：当本手 `drawsUsedThisHand === 0`（未点过补牌）时，倍率池 `+value`（受 `accumulateCap`），并将**叠池后的整池**并入当次额外倍率（与「非 hold 的 `accumulate_multiplier`」记账方式一致；见 `skillEngine.ts`）。
- `random_hand_add_multiplier`：`randomMultMin`～`randomMultMax` **闭区间均匀整数**（当前 **`random_double_hand` 为 [2, 15]**，缺省回退见 `skillEngine.rollRandomHandAddMultiplier` / `skillCardFaceStats`）。骰点在 **`flipCards`（翻牌 deal→hold）** 写入 `run.pendingRandomHandMult`，本手内 Hold 预览与结算一致；`dealInitialHand` 发新一手时清除。若本手已翻牌后**才**在商店购入该技能，则在购入当刻补骰；结算时若仍无缓存则回退当场 `roll`（防异常路径）。
- `per_remaining_hand_add_multiplier`：`value` 为「每还剩 1 手未打」的倍加数；计分时刻 `usedHands` 尚未计入本手，**剩余手数** = `max(0, stageTotalHands - stageUsedHands - 1)`，本手加性倍率 = 该值 × `value`。
- `per_run_diamond_score` / `per_run_diamond_multiplier`：贡献 = `floor(局内💎取样 / N) * value`，`N` 为可选 **`perRunDiamondsCost`（缺省 1）**；取样与 `skillEngine` 一致（不含本手待发钻）。**钻石加倍** 当前为 **N=3**、**value=1**（每 3💎 +1 倍）。

## 已拍板 / 说明（随机加倍 · 剩手加倍）

1. **随机加倍**：**翻牌即锁定**本手倍数（见上 `pendingRandomHandMult`）；Hold 预览与最终结算使用同一数值；随机为 **[2, 15] 均匀整数**（`runSkillPool.json`），不设权重/难度挂钩（除非日后改 JSON）。
2. **剩手加倍**：无限关**每一关** `totalHands` 仍由模板/词缀决定（如无限基底为 6 手再经 `getEffectiveStage`），**不是**「整段无限共用手数池」无限膨胀，故不按「无限关」单独削该技能。
3. **末手**：剩余尾手为 0 时 **+0 倍**，**不要**末手保底。
4. **`getEffectiveStage` 是什么意思**：主线/精英词缀可能**减少本关总手数**等；计分与 UI 都应基于**同一套生效关卡数据**。代码里计分用 `getEffectiveStage(rawStage, acquiredSkillIds)`；若某块 UI 误用**未套用词缀的 raw `stages[]`**，剩手倍率的「+N」可能与结算差 1 手——文档提醒：**手数取样与结算同源（effective）**。
5. **禁小丑（精英词缀）**：`runEliteModifiers.json` 中 `ban_jokers` 写入关卡 `banJokers: true`，本关 `injectJokers` 概率为 **0**；持有 **精英关无限制** 时与花色/牌型禁限等同被解除（`getEffectiveStage` 将 `banJokers` 置为 `false`）。
