# 响应式断点配置说明

## 📱💻 断点设置

### 当前配置
- **手机端（竖屏）**: `< 1320px`
- **PC 端**: `≥ 1320px`

## 🎯 Tailwind CSS 配置

在 `tailwind.config.js` 中，所有断点统一设置为 `1320px`：

```javascript
screens: {
  'sm': '1320px',   // ≥1320px 为 PC 端
  'md': '1320px',
  'lg': '1320px',
  'xl': '1320px',
  '2xl': '1320px',
}
```

## 📐 设计原则

### Mobile First（移动优先）

- **默认样式**（无前缀）= 手机竖屏样式（< 1320px）
- **`sm:` 前缀** = PC 端样式（≥ 1320px）

### 示例

```tsx
// 默认（手机）: w-20 (80px)
// PC端: sm:w-24 (96px)
className="w-20 sm:w-24"
```

## 🎮 适配策略

### 手机端（< 1320px）
- ✅ **16:9 竖屏优化**
- ✅ 紧凑布局，节省空间
- ✅ 较小的卡牌尺寸
- ✅ 触摸友好的按钮尺寸
- ✅ 堆叠式布局（flex-col）

### PC 端（≥ 1320px）
- ✅ 宽松布局，视觉舒适
- ✅ 较大的卡牌尺寸
- ✅ 横向布局（flex-row）
- ✅ 更多空白间距
- ✅ 悬停效果增强

## 🔄 自动适配

系统会根据浏览器窗口宽度自动切换样式：

1. **手机浏览器（通常 < 1320px）**
   - 自动应用手机端样式
   - 竖屏优化布局

2. **缩小的 PC 浏览器窗口（< 1320px）**
   - 也会切换到手机端样式
   - 确保在任何小窗口下都能正常显示

3. **PC 浏览器（≥ 1320px）**
   - 应用 PC 端样式
   - 充分利用大屏空间

## 📊 常用尺寸对照

| 元素 | 手机端 | PC 端 |
|------|--------|-------|
| **卡牌** | `w-20 h-28` (80×112px) | `sm:w-24 sm:h-36` (96×144px) |
| **图标** | `w-5 h-5` (20×20px) | `sm:w-6 sm:h-6` (24×24px) |
| **按钮** | `px-6 py-3 text-lg` | `sm:px-8 sm:py-4 sm:text-xl` |
| **间距** | `gap-2 py-4 mb-4` | `sm:gap-4 sm:py-8 sm:mb-8` |
| **容器** | `p-3` | `sm:p-4` |

- **PC**: http://localhost:5173/

### 3. 响应式测试点
- 1319px（手机样式）
- 1320px（PC 样式）
- 确认样式切换流畅

## 💡 注意事项

1. **所有新增样式都应遵循 Mobile First**
   ```tsx
   // ✅ 正确：默认手机，sm: 为 PC
   className="text-base sm:text-lg"
   
   // ❌ 错误：默认 PC，需要手动处理小屏
   className="text-lg"
   ```

2. **确保手机端可用性**
   - 按钮足够大（最小 44×44px）
   - 文字可读（最小 14px）
   - 间距合理（避免误触）

3. **避免固定宽度**
   - 使用 `max-w-*` 而非 `w-*`
   - 使用相对单位（rem, %, vw）
   - 确保内容自适应

## 🎨 最佳实践

```tsx
// 推荐的响应式类组合
<div className={clsx(
  // 基础样式（手机）
  "p-4 gap-2 text-base",
  // PC 端增强
  "sm:p-8 sm:gap-4 sm:text-lg"
)}>
  {/* 内容 */}
</div>
```
