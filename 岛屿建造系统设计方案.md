# 岛屿建造系统设计方案

## 一、系统概述

### 1.1 功能定位
在超级牌区域实现一个多场景悬空岛建造系统，包含52个不同的岛屿场景，每个场景对应一张超级牌。玩家使用游戏货币（$）购买并放置岛屿元素，完成场景建造即可解锁对应的超级牌。

### 1.2 核心玩法
- **52个场景**：每个场景对应一张超级牌（共52张）
- **元素建造**：每个场景包含1个底座（免费）+ 13个可购买元素
- **价格匹配**：每个场景13个元素的总价 = 该场景对应超级牌的价格
- **奖励机制**：完成场景建造立即解锁对应的超级牌
- **场景切换**：可以查看和建造任意已解锁的场景
- **长期目标**：激励玩家持续游戏获取货币，完成所有52个场景

### 1.3 场景与超级牌对应关系
- 场景按顺序对应超级牌的索引（0-51）
- 场景0 对应 第1张超级牌（$300）
- 场景1 对应 第2张超级牌（$500）
- ...
- 场景51 对应 第52张超级牌
- 使用 `getSuperCardFromIndex(sceneIndex)` 获取对应超级牌信息

## 二、资源分析

### 2.1 资源结构说明

#### 场景文件命名规则
```
img_scene{场景编号}_layer{层级编号}{类型标识}.png

示例：
- img_scene01_layer14N.png  (场景1的底座层)
- img_scene01_layer01E.png  (场景1的元素1)
- img_scene02_layer14N.png  (场景2的底座层)
- ...
```

**层级编号**：
- `layer14` = 底座层（Base，免费，每个场景必有）
- `layer01` ~ `layer13` = 13个可购买元素（每个场景都有）

**类型标识**：
- `N` = Normal/Base（基础层/底座）
- `E` = Element（核心元素）
- `D` = Decoration（装饰元素）
- `U` = Upper（上层元素）

#### 资源文件结构
- **场景目录**：`01/`, `02/`, `03/`, ... `52/`
- **每个场景包含**：
  - 1个底座文件：`img_scene{编号}_layer14N.png`
  - 13个元素文件：`img_scene{编号}_layer01{E/D/U}.png` ~ `img_scene{编号}_layer13{E/D/U}.png`
  - 1个Prefab文件：`Map{编号}.prefab`（包含坐标信息）

**当前示例场景（场景1）**：
- 底座：`img_scene01_layer14N.png` (937x750px)
- 13个元素：`layer01E.png` ~ `layer13D.png`
- 坐标文件：`Map1.prefab`

### 3.2 场景系统设计

#### 3.2.1 场景解锁规则
- **初始场景**：场景0（第1张超级牌）默认解锁
- **后续场景**：完成前一个场景后解锁下一个场景
- **例外**：如果超级牌已通过其他方式解锁，对应场景自动解锁（可选规则）

#### 3.2.2 单个场景建造规则
- **底座**：场景加载时自动显示（免费）
- **13个元素**：需要依次购买
- **购买顺序**：可以按任意顺序购买，无需前置条件
- **完成判定**：13个元素全部购买完成
- **完成奖励**：立即解锁对应的超级牌（调用 `gameStore.unlockSuperCard(superCardIndex)`）

#### 3.2.3 价格分配策略
每个场景的13个元素价格总和 = 该场景对应超级牌的价格

**价格分配算法**：
```typescript
function distributeElementPrices(totalPrice: number, elementCount: number = 13): number[] {
  // 方案1：均分（最简单）
  const basePrice = Math.floor(totalPrice / elementCount);
  const remainder = totalPrice % elementCount;
  
  const prices = Array(elementCount).fill(basePrice);
  // 将余数分配给前几个元素
  for (let i = 0; i < remainder; i++) {
    prices[i] += 1;
  }
  
  return prices;
  
  // 方案2：递增分配（推荐，更有挑战性）
  // 前几个元素便宜，后几个元素贵
  // 可以设置一个递增比例，如：1.1x, 1.2x, ..., 1.3x
}
```

**推荐方案：递增分配**
- 前3个元素：各占总价的 5%（快速进度反馈）
- 中间7个元素：各占总价的 10%（主要部分）
- 后3个元素：各占总价的 15%（最后冲刺）
- 总和验证：3×5% + 7×10% + 3×15% = 100% ✓

### 3.3 价格体系设计

#### 3.3.1 超级牌价格参考
根据 `src/utils/superCardPrices.ts`：
- 第1张超级牌：$300
- 第2张超级牌：$500
- 第3张超级牌：$800
- 第4张超级牌：$1,200
- 第5张超级牌：$1,700
- ...（价格递增规律：每次递增金额增加$100）
- 第52张超级牌：约 $140,000+

#### 3.3.2 场景元素价格分配示例

**场景0（对应第1张超级牌，总价$300）**：
```
元素1-3（前3个）：各 $15  (5% × $300 = $15)
元素4-10（中间7个）：各 $30 (10% × $300 = $30)
元素11-13（后3个）：各 $45 (15% × $300 = $45)
验证：3×$15 + 7×$30 + 3×$45 = $45 + $210 + $135 = $390 ❌

修正：使用实际均分
总计：$300 ÷ 13 ≈ $23.08
分配：
- 元素1-12：各 $23 ($23 × 12 = $276)
- 元素13：$24 ($276 + $24 = $300) ✓
```

**场景1（对应第2张超级牌，总价$500）**：
```
总计：$500 ÷ 13 ≈ $38.46
分配：
- 元素1-8：各 $38 ($38 × 8 = $304)
- 元素9-13：各 $39 ($39 × 5 = $195)
- 验证：$304 + $195 = $499，补1：元素1改为$39，总计$500 ✓
```

#### 3.3.3 价格计算工具函数
```typescript

 * 获取场景的总价（等于对应超级牌的价格）

/**
 * 计算场景中每个元素的价格

 */
function calculateElementPrices(sceneIndex: number): number[] {
  const totalPrice = getSceneTotalPrice(sceneIndex);
  const elementCount = 13;
  
  // 均分算法
  const basePrice = Math.floor(totalPrice / elementCount);
  const remainder = totalPrice % elementCount;
  
  const prices = Array(elementCount).fill(basePrice);
  
  // 将余数分配给前几个元素
  for (let i = 0; i < remainder; i++) {
    prices[i] += 1;
  }
  
  return prices;
}
```

#### 3.3.4 价格平衡说明
- **早期场景**：价格低（$300-$1,200），快速获得成就感
- **中期场景**：价格适中（$1,700-$10,000），稳定进度
- **后期场景**：价格高（$10,000+），长期目标

#### 4.3.1 Unity 到 Web 坐标转换
Unity 使用中心点锚定（0.5, 0.5），需要转换为 Web 绝对定位：

```typescript
function convertUnityToWeb(
  unityPos: { x: number; y: number },
  canvasSize: { width: number; height: number }
): { left: number; top: number } {
  // Unity 坐标：中心点为(0,0)，向右下为正
  // Web 坐标：左上为(0,0)，向右下为正
  const left = canvasSize.width / 2 + unityPos.x;
  const top = canvasSize.height / 2 - unityPos.y; // Y轴翻转
  
  return { left, top };
}
```

```typescript
      description: `场景 ${sceneIndex + 1} 的元素 ${layerId}`,
      price: elementPrices[i],
      position: prefabElement?.position || { x: 0, y: 0 },
      size: prefabElement?.size || { width: 100, height: 100 },
      layerType: i < 4 ? 'element' : i < 10 ? 'decoration' : 'upper',
      zIndex: layerId,
      isBase: false,
    });
  }
  
  // 构建场景配置
  return {
    sceneIndex,
    name: `场景 ${sceneIndex + 1}`,
    description: superCardInfo ? `对应 ${superCardInfo.suit} ${superCardInfo.rank}` : `场景 ${sceneIndex + 1}`,
    superCardIndex: sceneIndex,
    superCardPrice,
    elements: [baseElement, ...elements],
    baseElement,
    purchasableElements: elements,
    totalPrice: superCardPrice,
    prefabPath: `/${String(sceneIndex + 1).padStart(2, '0')}/Map${sceneIndex + 1}.prefab`,
  };
}

/**
 * 生成所有52个场景的配置（懒加载）
 */
const SCENE_CONFIGS_CACHE = new Map<number, Promise<IslandScene>>();

export function getSceneConfig(sceneIndex: number): Promise<IslandScene> {
  if (sceneIndex < 0 || sceneIndex >= 52) {
    throw new Error(`Invalid scene index: ${sceneIndex}`);
  }
  
  if (!SCENE_CONFIGS_CACHE.has(sceneIndex)) {
    SCENE_CONFIGS_CACHE.set(sceneIndex, generateSceneConfig(sceneIndex));
  }
  
  return SCENE_CONFIGS_CACHE.get(sceneIndex)!;
}
```

```typescript
// utils/islandConfig.ts

/**
 * 获取初始解锁的场景列表
 */
export function getInitialUnlockedScenes(): number[] {
  return [0]; // 场景0默认解锁
}

/**
 * 检查场景是否应该解锁
 */
export function shouldUnlockScene(sceneIndex: number, completedScenes: number[]): boolean {
  // 规则1：前一个场景已完成
  if (sceneIndex > 0) {
    return completedScenes.includes(sceneIndex - 1);
  }
  
  // 规则2：场景0始终解锁
  return sceneIndex === 0;
}
```
