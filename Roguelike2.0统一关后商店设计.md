> 状态：**策划定稿**（实现以代码为准；落地时同步 `rewardEngine.ts`、`RewardState`、`RewardModal`、`roguelikeStore` 及相关 md）

## 1. 目标

- **每关通关后只进入 1 次**统一商店界面（不再分「升级店 → 超级牌店」两步，也不再主线精英「连续两次技能店」）。
- 商店一次展示 **6 张卡**，由 **技能牌、升级牌、超级牌（属性牌）** 组成；**不按类型分栏分区**（UI 上不拆「技能区 / 升级区 / 超级牌区」列块，仅布局区分「购买区」与「卖出区」，见 §5）。
- **开局**：**取消**开局技能商店；**每局初始**持有 **3💎**（`runDiamonds` 初始为 **3**，实现以代码为准）。

## 2. 三位位型（技能 / 升级 / 超级）

三位数字 **固定顺序**为：

| 位 | 含义 |
|----|------|
| 第 1 位 | 本关商店中 **技能牌** 张数 |
| 第 2 位 | **升级牌** 张数 |
| 第 3 位 | **超级牌（属性牌）** 张数 |

三张数之和恒为 **6**。

### 2.1 普通关（非精英规则关）

以下 **3** 种位型各 **33⅓%**（合计 100%）：

| 位型 | 技能 | 升级 | 超级 |
|------|------|------|------|
| 222 | 2 | 2 | 2 |
| 231 | 2 | 3 | 1 |
| 132 | 1 | 3 | 2 |

### 2.2 精英关 / Boss 关（同一套规则）

以下 2 种位型各 **50%**（合计 100%）：

| 位型 | 技能 | 升级 | 超级 |
|------|------|------|------|
| 330 | 3 | 3 | 0 |
| 321 | 3 | 2 | 1 |

- **330**：本关无超级牌候选（仅技能 + 升级）；**321**：三类均有候选。按设计执行，不设「每类至少 1」保底。

### 2.3 前两关（k=1、k=2）附加定稿

以下仅针对 **通关第 1、2 关后**弹出的统一商店（**商店用关卡序号 k=1、k=2**，与 §3 一致）：

- **位型**：**固定 330**（不经 §2.1 随机池；亦不经 §2.2，因 k 不为 3 的倍数）。
- **技能品质**：三候选均为 **绿**（**绿绿绿** 模板；绿库存不足时按 `pickSkillsByQualityPattern` 既有回退补齐）。
- **技能边**：本两关商店内 **不出现带边**（强化均为 **normal**，不Roll银/金/彩/黑边）。

### 2.4 技能附加边（k≥3；统一商店技能候选）

> 实现：`rewardEngine.ts` 对「本批技能槽」**联合掷边**——**每批至多 1 张**技能为非 `normal`（银/金/彩/黑），带边落在哪一格 **均匀随机**。**k=1、k=2** 仍强制 §2.3，本条不生效。

| 场景 | 无附加 | 银 | 金 | 彩 | 黑 |
|------|--------|----|----|----|-----|
| **首次打开本关商店** | 88% | 4% | 4% | 2% | 2% |
| **本关 💎 刷新后**（整页重 Roll 技能边） | 82% | 6% | 5% | 4% | 3% |

- 上表为 **整批技能** 的互斥结果：例如「银 4%」表示本批技能里 **恰好一张** 为银边、其余为无附加，而非每卡独立 4%。
- 与「每关商店 1 次 💎 刷新」配合：首次用首行权重，点刷新后用次行权重；**每批仍至多一张带边**。

### 2.5 超级信用卡（`super_credit_card`）商店特例

- **通关前 5 关**对应的统一商店（**`shopStageK` = 1～5**，与 §3 一致）技能池中：**不出现**「超级信用卡」；**k ≥ 6** 起与其它技能同等进入随机池（仍受已拥有 / 本局已卖出等规则过滤）。
- 实现：`rewardEngine.ts` `generateSkillOptions`（含 💎 刷新重 Roll 同 `k`）。

### 2.6 黑边技能定向保底（扩技能槽；按“出现 vs 拥有”区分）

目标：在保留 §2.4 既有「银/金/彩/黑」掷边规则的前提下，**在特定关数后**对“黑边技能出现”进行**定向软保底**，以更稳定地扩展技能槽位（黑边=增加槽位，高价值，玩家默认倾向购买）。

#### 2.6.1 术语
- **商店出现黑边**：本关商店的**技能候选**中存在任意 `enhancement === black`（不要求购买；💎 刷新后出现也算）。
- **拥有黑边**：本局已装备技能里 `skillEnhancements[id] === black` 的数量（记为 `ownedBlackCount`）。

#### 2.6.2 门槛（`gateK(N)`）
当本关商店序号 `k` 满足下列条件时，启动对应“缺口段”的黑边保底：

- `ownedBlackCount <= 0` 且 `k >= 15`
- `ownedBlackCount <= 1` 且 `k >= 35`
- `ownedBlackCount <= 2` 且 `k >= 55`
- … 以此类推

统一写法：

\[
gateK(N) = 19 + 20N
\]

当 `ownedBlackCount <= N` 且 `k >= gateK(N)` 时，启用该段的“黑边出现保底”。

#### 2.6.3 概率曲线（10 关内提升到 100%）
以“每关进入一次商店”为一次机会（**每关一次**，不按 step 计数）。

记 `miss` 为当前段的“未出现黑边的商店次数”累积（只要该关商店**没有出现黑边**，则 `miss += 1`；若出现且购买则清零，见下条）。

- 基础概率 `p0 = 5%`
- 保底步数 `T = 10`

本关商店额外注入黑边的触发概率：

\[
p(miss) = \min(1,\; p0 + (1-p0)\cdot \frac{miss}{T})
\]

> 注：该“注入”只保证本关技能候选**至少 1 张**为黑边；其余候选仍走 §2.4 的边规则。

#### 2.6.4 注入规则（最多 1 张；随机落点；仅技能候选）
- **注入位置**：只对本关的**技能候选**生效（`skillOptions`）；升级/超级牌不参与。
- **最多 1 张**：每关商店最多保证 1 张黑边技能。
- **随机落点**：在技能候选中随机选 1 张注入为黑边（优先选 `normal` 槽，避免覆盖银/金/彩）。
- **若该关技能候选已自然出现黑边**（来自 §2.4 roll），则视为“已出现”，不再额外注入。

#### 2.6.5 “出现但未购买”的特殊规则（促使购买；不考虑额外极端情况）
由于黑边技能价值极高，默认玩家会倾向购买；但仍需在逻辑上区分“出现”与“拥有”。

当某关商店**出现黑边**但玩家**未购买**（商店结束后 `ownedBlackCount` 未增加）时：

- 进入 **2 关冷却期**：接下来的 **2** 关商店不强行注入黑边，仅保留 §2.4 的默认掷边概率；
- 该段的 `miss` **仍继续累加**（不清零）；
- 到第 **3** 关（冷却结束后），按累加后的 `p(miss)` 再次尝试注入黑边，期望用户购买。

## 3. 何关算「精英关」（商店用）

- **k=1、k=2**：使用 **§2.3**（固定 **330** + 绿绿绿 + 技能无带边）；**不**使用 §2.1。
- **仅**当 **k ≥ 3** 且 **`k % 3 === 0`** 时，本关通关后的商店使用 **§2.2 精英/Boss 位型池**（330 / 321 各 50%）。
- **其余关卡**（k ≥ 3 且非 3 的倍数，如第 4、5、7… 关）使用 **§2.1 普通关位型池**。
- **Boss 关**：与精英关 **相同**——只要满足上述 **`k % 3 === 0`**，即用精英池；**不**再单独依赖 `runStageTemplates.json` 里的 `isBoss` / `isElite` 切换商店类型（模板仍可用于词缀、目标金等；**商店分池仅以 `k%3===0` 为准**）。
- **无限模式**：与主线 **同一规则**——按无限关序从 1 计（或与主线衔接的连续序号），**序号为 3 的倍数**时用 §2.2，否则 §2.1（实现时需在 `generateRewardForStage` 或等价入口统一 `k` 定义并写注释）。

> 说明：此前代码曾用「组内 `groupIndex % 3`」区分步进；本定稿改为 **显式 `k % 3 === 0`**，避免与旧「每 3 关一组里第 3 格」混读。

## 4. 生成与规则（沿用现有引擎）

- **技能 / 升级 / 超级** 的候选生成逻辑 **全沿用** 当前 `rewardEngine`（及关联池、去重、已拥有技能不出现、`k≤2` 强制无边等）——**仅**把「每次抽 3 张」扩展为按位型抽 **0～3 张**（某类为 0 时该类型本轮无候选）。技能附加边见 **§2.4**；**超级信用卡** 见 **§2.5**（k=1～5 不入池）。
- **钻石刷新**：**6 张一并刷新**（整页替换）；次数与成本沿用当前「每阶段 1 次 💎 刷新」思路，合并后等价于 **每关商店 1 次刷新**（具体字段名以实现为准）。

---

## 5. UI 布局（手机竖屏；不分栏分区）

- **不按类型分栏**：不在 UI 上用「左技能 / 中升级 / 右超级」等 **分区标题或独立栏** 区分类型；三类卡在同一视觉区域内，用牌种自身组件区分即可。
- **购买区**：**两行三列**（2×3）排布 **6 张待购卡**（技能 / 升级 / 超级混合，顺序以实现为准，可固定为行优先填充）。
- **卖出区**：**横向一行**展示 **已拥有技能牌**；**超过 5 张时可互相遮挡**（与关卡内技能条叠放意图一致）。
- **商店内包含已有技能**：已拥有技能 **始终可在卖出区展示并随时卖出**；**购买新技能** 的前提仍是 **有空槽位**（含黑边、`getEffectiveSkillSlotCap` 等现有规则）。

## 6. 流程节奏

- **每关结束后** → **仅 1 次**进入统一商店 → 玩家可买 0～多件 / 卖技能 / 💎 刷新（1 次）→ **继续**进入下一关。
- 不再存在：关内二次弹出商店、或普通关「先升级店再超级牌店」的两步流程。
