### 2. 十连抽系统 ✨✨✨

**消耗：** 1000 积分代币（相当于 10 次单抽）

**触发条件：** 拥有 ≥1000 积分

**操作方式：** 点击"十连抽 ($1000)"按钮

#### 视觉设计

**按钮样式：**
- 紫色边框 + 紫色文字
- 脉冲呼吸动画
- 两侧闪烁的 Sparkles 图标
- 显示保底进度提示

**结果展示：**
- 全屏弹窗展示 10 张卡牌
- 5×2 网格布局
- 卡牌依次飞入（每张延迟 0.1 秒）
- 顶部显示品质统计（绿色×N、蓝色×N、紫色×N）

#### 特殊效果

**抽到紫卡时：**
- 紫色卡牌带有紫色光环和脉冲动画

**操作提示：**

- 底部提示"点击右上角关闭或按 ESC 键"

### 3. 保底机制 🎯

#### 保底规则

**核心机制：** 30 抽必出 1 张紫色品质卡牌

**计数逻辑：**
- 每次抽卡（单抽或十连抽）后计数器 +1
- 抽到紫卡后计数器重置为 0
- 第 30 抽如果未出紫卡，强制出紫卡

**保底类型：** 硬保底（强制触发）

#### 保底进度显示

**位置：** 十连抽按钮下方

**显示内容：**
- 距离保底 ≤10 抽时：`距保底 X 抽 🎯`
- 距离保底 >10 抽时：`30抽保底紫卡`

**示例：**
```
第 1-19 抽：显示"30抽保底紫卡"
第 20 抽：显示"距保底 10 抽 🎯"
第 25 抽：显示"距保底 5 抽 🎯"
第 30 抽：必出紫卡，计数器重置
```

#### 数据持久化

**保存内容：** 保底计数器 `drawsSincePurple`

**存储位置：** localStorage（自动保存）

**刷新页面：** 保底进度不会丢失

## 💰 经济分析

### 单抽 vs 十连抽

| 项目 | 单抽 | 十连抽 |
|------|------|--------|
| 消耗 | 100 积分 | 1000 积分 |
| 张数 | 1 张 | 10 张 |
| 效率 | 标准 | 标准 |
| 体验 | 单张展示 | 批量展示 |
| 保底 | 30 抽 | 3 次十连 |

**注意：** 十连抽没有额外折扣，但提供更好的视觉体验和批量查看功能。

### 保底价值分析

**保底成本：** 30 × 100 = 3000 积分

**保底收益：** 至少 1 张紫卡（价值极高）

**期望分析：**
- 不保底：约 20 抽出 1 张紫卡（5% 概率）
- 有保底：最多 30 抽出 1 张紫卡
- 保底提升：降低了最坏情况，提升了抽卡确定性

**实际表现：**
- 运气好：5-10 抽就出紫卡
- 运气一般：15-20 抽出紫卡
- 运气差：25-30 抽出紫卡（保底生效）

## 🎯 策略建议

### 抽卡策略

#### 1. 新手阶段（0-50 次翻牌）

**建议：** 单抽为主
- 积分有限，优先保证单抽机会
- 积累绿色和蓝色品质卡牌
- 观察保底进度

#### 2. 成长阶段（50-200 次翻牌）

**建议：** 单抽 + 十连抽混合
- 积分较充裕时使用十连抽
- 关注保底进度，临近保底时优先十连抽
- 积累紫色卡牌

#### 3. 成熟阶段（200+ 次翻牌）

**建议：** 十连抽为主
- 积分富裕，追求效率和体验
- 定期十连抽，快速扩充紫色卡牌
- 保持牌池强度

### 保底利用技巧

#### 技巧 1：保底倒计时

**方法：**
1. 观察保底进度（十连抽按钮下方）
2. 距离保底 ≤10 抽时，优先十连抽
3. 确保在保底前完成抽卡

**效果：** 最大化保底价值，不浪费运气

#### 技巧 2：积分管理

**方法：**
1. 预留 3000 积分作为保底资金
2. 超过 3000 积分的部分用于十连抽
3. 确保不会因积分不足错过保底

**效果：** 稳定获取紫色卡牌

#### 技巧 3：集中抽卡

**方法：**
1. 积累 5000-10000 积分
2. 连续进行 5-10 次十连抽
3. 批量获取大量卡牌

**效果：** 
- 触发 1-3 次保底
- 获得 3-5 张紫色卡牌
- 大幅提升牌池强度

## 🔧 技术实现

### 保底计数器

```typescript
interface GameState {
  drawsSincePurple: number; // 距离上次紫卡的抽数
  // ...
}
```

### 单抽逻辑

```typescript
drawCard: () => {
  const { drawsSincePurple } = get();
  
  // 保底检查
  const isGuaranteedPurple = drawsSincePurple >= 29;
  
  let quality: CardQuality;
  if (isGuaranteedPurple) {
    quality = 'purple'; // 强制紫卡
  } else {
    // 正常概率
    const roll = Math.random();
    quality = roll < 0.75 ? 'green' 
            : roll < 0.95 ? 'blue' 
            : 'purple';
  }
  
  // 更新计数器
  set({
    drawsSincePurple: quality === 'purple' ? 0 : drawsSincePurple + 1
  });
  
  return newCard;
}
```

### 十连抽逻辑

```typescript
draw10Cards: () => {
  const drawnCards: Card[] = [];
  
  // 执行 10 次单抽
  for (let i = 0; i < 10; i++) {
    const card = drawCard();
    if (card) {
      drawnCards.push(card);
    }
  }
  
  return drawnCards;
}
```
