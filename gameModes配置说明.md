# 玩法配置说明文档

本文档对应 `src/config/gameModes.json`，用于方便查看和修改所有 52 个超级牌解锁玩法配置。

## 配置字段说明

- **玩法ID (id)**: 唯一标识符，用于代码中引用
- **玩法名称 (name)**: 显示给玩家的玩法名称
- **超级牌ID (superCardId)**: 对应的超级牌标识（格式：`spades_A`, `hearts_2` 等）
- **卡牌数量 (cardCount)**: 3、4 或 5 张
- **牌池类型 (cardPool)**: `"all"` 表示全部牌池（包含所有品质牌）
- **Joker概率 (jokerProbability)**: 0 或 0.08（小数形式）。JOKER+ 玩法为 8%，其他玩法为 0%
- **额外规则 (extraRules)**: 数组，可能包含：
  - `n_card_straight`: N张成顺（value: 3 或 4）
  - `n_card_flush`: N张成花（value: 3 或 4）
  - `color_flush`: 颜色成花（color: "red" 或 "black"）
  - `odd_straight`: 单数成顺
  - `even_straight`: 双数成顺
  - `multiplier_bonus`: 赔率加成（value: 加成倍数, handType: 牌型）
- **任务 (task)**: 
  - `type`: 任务类型（high_card, one_pair, two_pairs, three_of_a_kind, straight, flush, full_house, four_of_a_kind, five_of_a_kind, straight_flush, royal_flush, odd_straight, even_straight）
  - `target`: 目标次数
- **解锁费用 (cost)**: 从 `superCardUnlocks.json` 中获取，一次性支付金额

## 额外规则详细说明

### 1. N张成顺/成花
- **`n_card_straight`**: 允许 N 张牌组成顺子（value: 3 或 4）
- **`n_card_flush`**: 允许 N 张牌组成同花（value: 3 或 4）

**重要说明：**
- **超级3张玩法**：只发 3 张牌（`cardCount: 3`），通过额外规则判断这 3 张牌是否能组成 3 张顺子或 3 张同花
  - 如果满足 3 张顺子 → 识别为顺子（倍数 5）
  - 如果满足 3 张同花 → 识别为同花（倍数 7，优先级更高）
  - 只计算参与牌型的 3 张牌
  
- **超级4张玩法**：只发 4 张牌（`cardCount: 4`），通过额外规则判断这 4 张牌是否能组成 4 张顺子或 4 张同花
  - 如果满足 4 张顺子 → 识别为顺子（倍数 5）
  - 如果满足 4 张同花 → 识别为同花（倍数 7，优先级更高）
  - 只计算参与牌型的 4 张牌

**实现逻辑：**
1. 发牌阶段：根据 `cardCount` 只发对应数量的牌（3 张或 4 张）
2. 标准评估：3 张只能判断三条、一对、高牌；4 张只能判断四条、三条、两对、一对、高牌
3. 额外规则覆盖：通过 `n_card_straight` 和 `n_card_flush` 规则，判断是否能组成顺子或同花
4. 计分规则：只计算参与牌型的 n 张牌，基础倍数顺子=5、同花=7

### 2. 颜色成花
- **`color_flush`**: 限制同花的花色
  - `color: "red"`: 仅红桃(♥️)和方块(♦️)可成同花
  - `color: "black"`: 仅黑桃(♠️)和梅花(♣️)可成同花

### 3. 单数/双数成顺
- **`odd_straight`**: 单数成顺（A、3、5、7、9 等单数点数可成顺，普通顺子也算）
- **`even_straight`**: 双数成顺（2、4、6、8、10 等双数点数可成顺）

### 4. 赔率加成
- **`multiplier_bonus`**: 特定牌型赔率加成
  - `value`: 加成倍数（如 1 表示赔率+1）
  - `handType`: 适用的牌型（two_pairs, three_of_a_kind, straight, flush, full_house）

## 修改指南

### 修改玩法名称
在 `gameModes.json` 中找到对应的玩法，修改 `name` 字段。

### 修改任务目标
在 `superCardUnlocks.json` 中找到对应的超级牌，修改 `unlockConditions.task.target` 字段。

### 修改解锁费用
在 `superCardUnlocks.json` 中找到对应的超级牌，修改 `unlockConditions.cost` 字段。

### 修改 Joker 概率
在 `gameModes.json` 中找到对应的玩法，修改 `jokerProbability` 字段（0、0.03、0.08、0.1 等）。

### 修改额外规则
在 `gameModes.json` 中找到对应的玩法，修改 `extraRules` 数组。

### 修改卡牌数量
在 `gameModes.json` 中找到对应的玩法，修改 `cardCount` 字段（3、4 或 5）。

## 注意事项

1. **所有玩法都使用 "all" 牌池**（全部牌池），包含所有品质的牌
2. **Joker 概率是固定值**：
   - **JOKER+ 玩法**：统一为 8%（0.08）
   - **其他玩法**：统一为 0%（0）
3. **3 张和 4 张玩法无法完成 5 张牌成就**（顺子、同花、同花顺、皇家同花顺）
4. **解锁顺序固定**：从 ♠️2 开始，按花色和数值顺序（♠️2 → ♠️3 → ... → ♠️A → ♥️2 → ... → ♦️A）依次解锁
5. **价格递增规律**：每次递增金额增加$100
   - 第1张（♠️2）：$300
   - 第2张（♠️3）：$500（+$200）
   - 第3张（♠️4）：$800（+$300）
   - 第4张（♠️5）：$1,200（+$400）
   - 第5张（♠️6）：$1,700（+$500）
   - 第6张（♠️7）：$2,300（+$600）
   - ...以此类推
   - 第52张（♦️A）：$138,000
6. **JOKER+ 玩法任务**：所有 JOKER+ 玩法中，原"两对"任务已改为"三条"任务，目标统一为 5 次
7. **修改后需要重新构建**：修改 JSON 文件后，需要运行 `npm run build` 重新构建项目
