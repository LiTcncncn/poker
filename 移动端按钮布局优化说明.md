# 移动端按钮布局优化说明

## 🎯 优化目标

**重新组织移动端主界面按钮布局，提升操作效率和视觉体验**

## 📊 优化前的问题

### 原布局结构

```
┌──────────────────────────────┐
│  💰金币  ⚡体力  [图标按钮]   │ ← Header
├──────────────────────────────┤
│                              │
│    [牌型显示区域]            │
│                              │
│  [🎴] [🎴] [🎴] [🎴] [🎴]   │ ← 5张牌
│                              │
├──────────────────────────────┤
│  [翻牌] [单抽] [十连抽] [自动] │ ← 所有按钮挤在一起
└──────────────────────────────┘
```

**问题**：
1. ❌ 所有4个按钮在移动端会换行显示
2. ❌ 翻牌作为主操作，没有突出显示
3. ❌ 抽卡按钮和翻牌按钮混在一起，逻辑不清晰
4. ❌ 自动按钮与翻牌按钮分离，不便于快速切换

## ✅ 优化后的布局

### 新布局结构（移动端）

```
┌──────────────────────────────┐
│  💰金币  ⚡体力  [图标按钮]   │ ← Header
├──────────────────────────────┤
│   [单抽 $100]  [十连抽]      │ ← 抽卡按钮区（新增）
├──────────────────────────────┤
│                              │
│    [牌型显示区域]            │
│                              │
│  [🎴] [🎴] [🎴] [🎴] [🎴]   │ ← 5张牌
│                              │
├──────────────────────────────┤
│  [    翻牌/下一手   ] [自动] │ ← 游戏控制区
│  ←─────70%────→  ←─30%─→    │   (翻牌70%, 自动30%)
└──────────────────────────────┘
```

**优势**：
1. ✅ 功能分区明确：抽卡区 vs 游戏控制区
2. ✅ 翻牌按钮占据主导（70%宽度）
3. ✅ 自动按钮紧邻翻牌，便于快速切换
4. ✅ 抽卡按钮靠近金币显示，逻辑清晰
5. ✅ 无需换行，一屏显示所有操作

---

## 📝 详细实现

### 1️⃣ 抽卡按钮区（Header 下方）

#### 位置和显示
```tsx
{/* Draw Cards Section - 移动端在这里显示抽卡按钮 */}
<div className="w-full max-w-6xl mb-4 sm:hidden">
```

**特点**：
- 仅在移动端显示 (`sm:hidden`)
- 位于 Header 下方，金币体力信息下方
- 左右各留 `px-2` 边距

#### 按钮布局
```tsx
<div className="flex gap-2 px-2">
  <button className="flex-1">单抽 ($100)</button>
  <button className="flex-1">十连抽</button>
</div>
```

**特点**：
- 两个按钮均分宽度 (`flex-1`)
- 间距 `gap-2` (8px)
- 紧凑的文字和图标

#### 单抽按钮
```tsx
<button className="flex-1 px-4 py-3 rounded-xl font-bold text-base">
  <Sparkles className="w-4 h-4" />
  单抽 ($100)
</button>
```

**样式特点**：
- 绿色边框 (`border-green-500`)
- 文字 `text-base` (16px)
- 小图标 `w-4 h-4` (16px)
- 横向排列 `flex items-center`

#### 十连抽按钮
```tsx
<button className="flex-1 px-4 py-3 flex flex-col">
  <div className="flex items-center gap-1.5">
    <Sparkles className="w-4 h-4" />
    <span>十连抽</span>
    <Sparkles className="w-4 h-4" />
  </div>
  <span className="text-xs">距保底 X 抽</span>
</button>
```

**样式特点**：
- 紫色边框 (`border-purple-500`)
- 双行布局 (`flex-col`)
- 主文字 + 保底提示
- 小间距 `gap-0.5`

---

### 2️⃣ 游戏控制区（卡牌下方）

#### 移动端布局（翻牌70% + 自动30%）

```tsx
{/* 移动端：翻牌和自动按钮一行 */}
<div className="flex gap-2 sm:hidden mb-2">
  <button className="flex-[7]">翻牌</button>
  <button className="flex-[3]">自动</button>
</div>
```

**特点**：
- 仅移动端显示 (`sm:hidden`)
- `flex-[7]` : `flex-[3]` = **70% : 30%**
- 间距 `gap-2` (8px)

#### 翻牌按钮（主按钮）
```tsx
<button className="flex-[7] py-3 rounded-xl font-bold text-lg">
  {isFlipped ? "下一手" : "翻牌"}
</button>
```

**样式特点**：
- 占据 **70% 宽度** (`flex-[7]`)
- 蓝色渐变背景
- 大字体 `text-lg` (18px)
- 充足的纵向空间 `py-3`

#### 自动按钮（辅助按钮）
```tsx
<button className="flex-[3] py-3 rounded-xl font-bold text-base">
  {isAutoFlipping ? "⏸️ 停止" : "▶️ 自动"}
</button>
```

**样式特点**：
- 占据 **30% 宽度** (`flex-[3]`)
- 黄色边框（激活时）
- 中字体 `text-base` (16px)
- Emoji 图标节省空间

---

### 3️⃣ PC 端布局（保持原样）

#### 布局方式
```tsx
{/* PC端：所有按钮横向排列 */}
<div className="hidden sm:flex gap-6 justify-center flex-wrap">
  <button>翻牌</button>
  <button>单抽</button>
  <button>十连抽</button>
  <button>自动</button>
</div>
```

**特点**：
- 仅 PC 端显示 (`hidden sm:flex`)
- 横向排列，居中显示
- 宽松间距 `gap-6` (24px)
- 可以换行 (`flex-wrap`)
- 所有按钮保持原大小

```
│  ←─────70%────→ ←30%→     │
└────────────────────────────┘
```
- ✅ 分两个区域，逻辑清晰
- ✅ 翻牌按钮占主导（70%）
- ✅ 每个区域仅占一行
- ✅ 整体更紧凑

## 💡 交互优化

### 1. 视觉反馈

所有按钮都有：
```tsx
className="transition-all transform active:scale-95"
```
- 点击缩小效果 (`active:scale-95`)
- 平滑过渡 (`transition-all`)

### 2. 禁用状态

#### 翻牌按钮
```tsx
disabled={flipsRemaining <= 0 && !isFlipped}
// 无体力且未翻牌时禁用
```
- 灰色背景
- 禁用鼠标

#### 抽卡按钮
```tsx
disabled={money < 100}  // 单抽
disabled={money < 1000} // 十连抽
```
- 灰色边框
- 文字变暗

### 3. 状态指示

#### 自动按钮
```tsx
{isAutoFlipping ? "⏸️ 停止" : "▶️ 自动"}
```
- 播放/暂停 Emoji
- 背景色切换
- 边框颜色变化

#### 十连抽保底
```tsx
{drawsSincePurple >= 20 
  ? `距保底 ${30 - drawsSincePurple} 抽` 
  : '30抽保底'}
```
- 动态显示保底进度
- 接近保底时突出显示

## 🎯 优化成果

### ✅ 功能性改进
1. **逻辑分区**：抽卡操作与游戏控制分离，功能界限清晰
2. **操作效率**：主操作（翻牌）占据70%宽度，更易点击
3. **视觉层次**：按钮大小和位置反映重要性
4. **空间利用**：移动端无需换行，节省屏幕空间

### ✅ 视觉改进
1. **突出主操作**：翻牌按钮占据主导视觉位置
2. **颜色语义**：不同操作使用不同颜色，直观易懂
3. **布局对称**：抽卡区和控制区都是一行，视觉平衡
4. **间距统一**：所有区域使用 8px/24px 标准间距

### ✅ 交互改进
1. **快速切换**：自动按钮紧邻翻牌，便于快速开启/关闭
2. **逻辑清晰**：抽卡靠近金币，游戏控制靠近卡牌
3. **状态反馈**：所有按钮都有明确的禁用/激活状态
4. **保底提示**：十连抽显示保底进度，提升吸引力
