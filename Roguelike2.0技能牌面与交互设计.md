# Roguelike 2.0：技能卡「扑克牌形态」与交互设计

> 状态：**已确认**（银/金/彩/黑四档附加边与槽位规则已落地代码，视觉仍可迭代）  
> 基准：`roguelike/src/shared/components/Card.tsx` + `CardFaceV2.tsx` 的牌轮廓与牌面规格。

---

## 1. 目标

- 技能在 UI 上从「圆角矩形信息卡」改为与**局内扑克牌**在**外轮廓、圆角、宽高比**上**完全一致**的「技能牌」。
- 玩家可见用语：**银边 / 金边 / 彩边 / 黑边**（与旧称对应关系见 §2）；银金彩的**局内数值规则**不变；黑边仅影响**本局技能槽位**（见 §2、§4）。
- **品质稀有度**：继续用**牌面底色**区分绿 / 蓝 / 紫（与 `CardFaceV2` 绿蓝紫壳色同一套或刻意对齐的渐变/纯色）。
- **特殊边类型**：用**厚描边 / 特效**区分三种附加边（见 §4）；**无附加属性**时不展示特殊边文案区、**不使用**厚描边/流光，仅保留与普通牌一致的细边/壳（你已确认）。
- **交互**：点击技能牌**牌面**→ 弹出**技能详细信息**（见 §6）；与「购买 / 卖出」等按钮的点击域分离，避免误触。

## 2. 术语迁移（玩家可见 ↔ 旧称 ↔ 代码）

| 玩家可见（定稿） | 旧玩家/文档用语 | 局内效果 | 代码枚举 |
|------------------|-----------------|----------|----------|
| **银边**         | 闪              | 每手额外 +$30（加算层） | `flash`        |
| **金边**         | 金边            | 每手额外 +10 倍（倍率层） | `gold`      |
| **彩边**         | 镭射            | 每手额外 ×1.5（独立乘区） | `laser`     |
| **黑边**         | （新增）        | 该技能**上阵后**本局总技能槽 **+1**（按枚线性叠加；与扩容等来源可叠；见版本规划稿槽位公式） | `black`     |
| （无）           | 普通 / normal   | 无附加           | `normal`            |

说明：

- **策划与 UI 文案**统一使用「银边 / 金边 / 彩边」。
- **代码层**使用 `SkillEnhancement = 'normal' | 'flash' | 'gold' | 'laser' | 'black'`，通过 **display map**：`flash→银边`、`gold→金边`、`laser→彩边`、`black→黑边`。
- **战斗 / 结算日志**（如 `skillEngine` 里 `skillName` 后缀）：与玩家可见称呼一致，使用 `[银边]` / `[金边]` / `[彩边]` 后缀；黑边无本手数值加成，一般不缀数值后缀（槽位在 run 层统计）。

生成概率、商店加价、卖出返还等**数值**仍以 `Roguelike2.0版本更新规划（草案）.md` 为准。**黑边**：商店加价 **`+4💎`**（与金边加价同级），卖出返还 **`+2💎`**（与金边返还同级）。**统一商店技能边**：每批至多 1 张带边；首次 88/4/4/2/2%（无/银/金/彩/黑），💎 刷新后 82/6/5/4/3%；**k=1、2** 仍全无边（见 `Roguelike2.0统一关后商店设计.md` §2.4）。

## 3. 与扑克牌一致的规格（硬约束）

对齐来源：`Card.tsx` 中外层尺寸与牌体圆角逻辑。

- **宽高比**：`aspect-[2/3]`（与默认扑克组件一致）。
- **默认尺寸参考**：与牌相同——**宽度随父级 / flex 分配**，不用 `vw` 把技能牌宽绑到整屏视口；在商店/弹层中可按容器缩放，但**比例固定 2:3**（与主应用 `max-w-[390px]` 版心一致，见仓库 `.cursor/rules/mobile-ui-only.mdc`）。
- **圆角**：与**非 super** 扑克牌面一致：`rounded-[14px]`；技能牌不按超级牌 quality 切换圆角。
- **外框/阴影**：外层可与 `CardFaceV2` 牌壳类似的 `border` + 轻阴影，使技能牌与手牌区扑克剪影一致。

## 4. 牌面元素（信息架构）

1. **技能名**  
   - 主标题，字号随牌宽缩放（可用 `cqw` 容器查询）。

2. **数值汇总行（牌面）**  
   - **不在牌面做结构化数值解析**；牌面仅保留必要静态展示（如附加边一行，见下）。**具体数字拆解、动态累积说明均在详情弹层内解析与展示**（你已确认）。

3. **特殊边数值（仅当有附加属性时）**  
   - **特殊边标识 + 数值（或规则摘要）**  
   - 例：`银边 +$30`、`金边 +10 倍`、`彩边 ×1.5`、`黑边 技能+1`（黑边无底条「倍率数字」，仅规则摘要）  
   - `normal`：**不出现**本区域。

4. **品质稀有度**  
   - **仅用牌面底色**表达绿 / 蓝 / 紫。

5. **特殊边视觉**  
   - **银边**：建议 **银灰金属感厚描边**（对应 `flash`，实施先试版，hex/粗细你后续再改）。  
   - **金边**：**金色厚描边**。  
   - **彩边**：对应 `laser`，**彩色流光**表现（可与厚描边结合或以内圈流光为主；实施先试版，你后续再验收微调）。  
   - **黑边**：**黑灰金属感环带 + 表层灰白圆斑**微动（与银金彩区分；`prefers-reduced-motion` 下可关动画）。  
   - `normal`：**无**厚描边、**无**流光。

6. **长描述**  
   - 不进牌面或仅一行省略；完整描述进详情。

## 5. 出现场景（统一组件）

下列场景复用同一「技能牌」组件，`variant` 控制尺寸与交互：

- 关后商店候选（`RewardModal`）
- 已拥有技能区（卖出按钮在牌外或牌下方，避免与点牌详情冲突）
- **关卡内技能区**：改为**迷你 2:3 技能牌**横向排列；若宽度不够则**互相遮挡一部分**，布局意图对齐 **vpoker「我的牌池」固定区**（你已确认）。

---

## 6. 点击详细逻辑

- **点击牌面主体**：打开详情弹层 / 底部抽屉。
- **详情内**：完整描述、触发条件、以及**在此处做数值与机制的解析展示**（含动态累积等）；牌面不要求解析（你已确认）。
- **商店场景**：详情弹层内是否增加「购买」等入口——**待技能牌样式定稿后由你再设计**（你已确认）；关闭详情后仍停留在商店。
- **卖出**：仍走现有二次确认；不属于「点牌详情」路径。

## 8. 已确认结论汇总（原提问闭环）

| 议题 | 结论 |
|------|------|
| 日志后缀 | `[银边]` / `[金边]` / `[彩边]`，与牌面展示一致；黑边无本手数值后缀 |
| 黑边槽位 | 总槽位 = 基础 `skillSlotCap`（默认 **5**）+ **已持有**黑边枚数；牌面「技能+1」指该枚黑边带来的 **1 格上限增量即其自身所占格**，不会对分母「入队 + 黑边效果」各 +1 变成 `5/7`；例 **`4/5`** 购入并上阵 1 枚黑边 → **`5/6`**。HUD 仅在 **「技能」标签角标** 显示 `n/有效槽`（如 `1/5`），牌面容器内不再重复一行。 |
| 黑边经济 | 商店加价 **+4💎**、卖出 **+2💎**；统一商店技能边见定稿 §2.4（每批至多一张带边 + 首刷/刷新两套权重） |
| 关卡内技能条 | 迷你 2:3 技能牌，可重叠，参考 vpoker 牌池固定区 |
| 商店详情内购买 | 待定，技能牌样式确定后再设计 |
| 厚描边 / 色值 / 流光强度 | 实施先试版，完成后你再改 |
| 牌面数值 | 牌面不解析；详情里解析 |
| 无附加属性 | 不显示特殊边区、无厚描边/流光 |

## 9. 实施备注（与代码对齐）

- **`SkillPlayingCard`**：外壳与 `Card` 同比例，内容为技能布局；四档附加边含黑边环带与底条文案。
- `RewardModal`：技能候选 + 购买键；点牌打开 `SkillPlayingCardDetailModal`；**有效槽位**与购入「黑边」候选时的临时 +1 与 `roguelikeStore.chooseSkill` 一致。
- 关卡内：迷你技能牌叠放见 `SkillPanel` / `StageView`。
- 详情：Modal 栈，注意移动端安全区与滚动。
