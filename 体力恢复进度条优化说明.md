# 体力恢复进度条优化说明

## 🎯 优化目标

**将体力进度条从显示总体力百分比改为显示单点体力的恢复进度**

## 📊 优化前的问题

### 原实现

```tsx
// 显示总体力百分比
style={{ width: `${(flipsRemaining / maxFlips) * 100}%` }}
```

**问题**：
- ❌ 显示的是总体力条的百分比（如 15/20 = 75%）
- ❌ 无法看到当前这1点体力的恢复进度
- ❌ 用户不知道还要等多久才能恢复下1点体力
- ❌ 进度条在体力未满时总是显示部分填充，不够直观

**示例**：
```
体力: 15/20
进度条: ████████████████░░░░  (75%)
```
- 用户看到75%的进度条，但不知道这代表什么
- 无法判断距离恢复下1点体力还有多久

---

## ✅ 优化后的实现

### 新实现

```tsx
// 显示单点体力的恢复进度 (0-100%)
style={{ width: `${recoveryProgress * 100}%` }}
```

**效果**：
- ✅ 显示当前这1点体力的恢复进度（0-100%）
- ✅ 每20秒恢复1点体力，进度条从0%到100%
- ✅ 进度条满了，体力+1，进度条重置为0%
- ✅ 用户可以清楚看到距离恢复还有多久

**示例**：
```
体力: 15/20
进度条: ████████░░░░░░░░░░░░  (40%)
```
- 用户看到40%的进度条，知道当前这1点体力已经恢复了40%
- 还有60%的时间（约12秒）就能恢复下1点体力

## 📝 技术实现

### 1️⃣ Store 层 - 添加进度计算函数

#### 文件: `src/store/gameStore.ts`

**新增接口**：
```typescript
getRecoveryProgress: () => number; // 获取当前恢复进度 (0-1)
```

**实现逻辑**：
```typescript
getRecoveryProgress: () => {
    const state = get();
    const now = Date.now();
    
    // 如果体力已满，进度条为0
    if (state.flipsRemaining >= state.maxFlips) {
        return 0;
    }
    
    const timeSinceLastRecover = now - state.lastRecoveryTime;
    const RECOVER_INTERVAL = 20000; // 20秒
    
    // 计算当前恢复进度 (0-1)
    const progress = Math.min(1, timeSinceLastRecover / RECOVER_INTERVAL);
    
    return progress;
}
```

**关键点**：
- 体力已满时返回 0（进度条为空）
- 计算从 `lastRecoveryTime` 到现在的时间差
- 除以 `RECOVER_INTERVAL` (20000ms) 得到进度百分比
- 使用 `Math.min(1, ...)` 确保不超过 100%

---

### 2️⃣ 组件层 - 实时更新进度条

#### 文件: `src/App.tsx`

**新增状态**：
```typescript
const [recoveryProgress, setRecoveryProgress] = useState(0);
```

**初始化进度**：
```typescript
useEffect(() => {
  setRecoveryProgress(getRecoveryProgress());
}, [getRecoveryProgress]);
```

**定期更新**：
```typescript
useEffect(() => {
  const interval = setInterval(() => {
    recoverEnergy();
    // 更新恢复进度
    setRecoveryProgress(getRecoveryProgress());
  }, 100); // 每100ms更新一次，让进度条更流畅
  return () => clearInterval(interval);
}, [recoverEnergy, getRecoveryProgress]);
```

**进度条显示**：
```tsx
<div 
  className="h-full bg-blue-500 transition-all duration-100" 
  style={{ width: `${recoveryProgress * 100}%` }} 
/>
```

**关键点**：
- 每 100ms 更新一次进度（比原来的 1000ms 更流畅）
- 使用 `transition-all duration-100` 让进度条平滑过渡
- 进度条宽度 = `recoveryProgress * 100%`
