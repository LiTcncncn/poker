# 腾讯云静态网站空白页面问题排查

## 问题：上传 dist 后访问空白

### 可能原因及解决方案

#### 1. 上传方式错误（最常见）

**错误方式**：上传了整个 `dist` 文件夹
```
存储桶根目录/
  └── dist/
      ├── index.html
      ├── assets/
      └── pokers/
```
这样会导致访问路径变成：`https://xxx.com/dist/index.html`，资源路径 `/assets/...` 找不到。

**正确方式**：上传 `dist` 文件夹**里面的内容**
```
存储桶根目录/
  ├── index.html
  ├── assets/
  │   ├── index-0c7QtiHj.css
  │   └── index-Bt7rPuNA.js
  └── pokers/
      └── ...
```

**解决方法**：
1. 删除存储桶中的 `dist` 文件夹
2. 进入 `dist` 文件夹，选择**里面的所有文件和文件夹**
3. 上传到存储桶的**根目录**（不是 dist 文件夹内）

---

#### 2. 静态网站托管未开启或配置错误

**检查步骤**：
1. 进入 COS 控制台 → 你的存储桶
2. 点击左侧"基础配置" → "静态网站"
3. 确认"静态网站"功能已开启
4. 确认配置：
   - **索引文档**：`index.html`
   - **错误文档**：`index.html`（SPA应用必须）
5. 保存配置

---

#### 3. 访问路径错误

**正确访问方式**：
- 使用静态网站提供的访问节点，格式：
  ```
  https://<bucket-name>-<appid>.cos.<region>.myqcloud.com
  ```
- **不要**使用存储桶的默认访问地址（格式不同）

**如何获取正确地址**：
1. 进入存储桶 → "基础配置" → "静态网站"
2. 找到"访问节点"或"访问地址"
3. 使用这个地址访问

---

#### 4. 文件权限问题

**检查**：
1. 进入存储桶 → "文件列表"
2. 检查 `index.html` 的访问权限
3. 确保所有文件都是"公有读"权限

**解决方法**：
- 如果使用 COS 控制台上传，默认权限应该是正确的
- 如果使用命令行工具，确保设置了正确的权限

---

#### 5. 浏览器控制台错误

**检查方法**：
1. 打开浏览器开发者工具（F12）
2. 查看 Console 标签页，看是否有错误
3. 查看 Network 标签页，看哪些资源加载失败

**常见错误**：
- `404 Not Found`：资源路径错误
- `CORS error`：跨域问题（通常不会出现在静态网站托管）
- `MIME type` 错误：文件类型识别问题

---

## 快速修复步骤

### 步骤1：确认文件结构
在本地检查 `dist` 目录结构：
```bash
cd dist
ls -la
```
应该看到：
- `index.html`（在根目录）
- `assets/` 文件夹
- `pokers/` 文件夹

### 步骤2：重新上传（正确方式）

**方式A：使用 COS 控制台**
1. 进入存储桶 → "文件列表"
2. 删除之前上传的 `dist` 文件夹（如果存在）
3. 点击"上传文件"
4. **重要**：选择 `dist` 文件夹**里面的内容**（不是 dist 文件夹本身）
   - 选择 `index.html`
   - 选择 `assets` 文件夹
   - 选择 `pokers` 文件夹
5. 上传到存储桶的**根目录**

**方式B：使用命令行**
```bash
# 进入 dist 目录
cd dist

# 使用 coscmd 上传（需要先配置）
coscmd upload -rs . /

# 或者使用腾讯云 CLI
# 需要先安装和配置
```

### 步骤3：验证文件结构
上传后，在 COS 控制台检查文件列表，应该看到：
```
存储桶根目录/
├── index.html          ← 在根目录
├── assets/
│   ├── index-0c7QtiHj.css
│   └── index-Bt7rPuNA.js
└── pokers/
    └── ...
```

### 步骤4：检查静态网站配置
1. 存储桶 → "基础配置" → "静态网站"
2. 确认已开启
3. 索引文档：`index.html`
4. 错误文档：`index.html`

### 步骤5：使用正确的访问地址
使用静态网站提供的访问节点，不是存储桶的默认地址。

---

## 验证清单

上传完成后，按以下清单检查：

- [ ] `index.html` 在存储桶根目录（不在 dist 文件夹内）
- [ ] `assets/` 文件夹在存储桶根目录
- [ ] `pokers/` 文件夹在存储桶根目录
- [ ] 静态网站托管已开启
- [ ] 索引文档设置为 `index.html`
- [ ] 错误文档设置为 `index.html`
- [ ] 使用静态网站访问节点访问
- [ ] 浏览器控制台无 404 错误
- [ ] 浏览器 Network 面板显示资源正常加载

---

## 如果还是空白

1. **清除浏览器缓存**：Ctrl+F5 强制刷新
2. **检查浏览器控制台**：查看具体错误信息
3. **检查 Network 面板**：看哪些资源加载失败
4. **尝试无痕模式**：排除浏览器扩展干扰

如果问题依然存在，请提供：
- 浏览器控制台的错误信息
- Network 面板中失败的请求
- 存储桶的文件结构截图
