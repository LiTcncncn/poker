## 3. 2.0 单局玩法改造方案

## 3.1 新货币：局内钻石（`💎`）

### 3.1.1 获取规则（你提出的基础结算，落地版）
- 每关通关后结算 `💎`：
  - 普通关：`+3`
  - 精英关：`+5`
  - 剩余手数奖励：仅两档
    - 剩 `2` 手：`+2`
    - 剩 `1` 手：`+1`
  - 剩余 hold 奖励：仅一档
    - 剩 `1` 次 hold：`+1`

### 3.1.2 结算时机
- 关卡胜利进入奖励界面前结算并入账。
- 失败即结束本局，且不结算该关 `💎`。

### 3.1.3 数据结构草案
- `RunState` 新增：
  - `runDiamonds: number`（当前持有）
  - `diamondsEarnedTotal: number`（统计）
  - `diamondsSpentTotal: number`（统计）

## 3.2 关后奖励改造：从“选1”到“可购买”

### 3.2.1 新流程（每次奖励阶段）
1. 系统生成 3 个候选（技能/升级/超级牌，按当前阶段类型）。
2. 显示每个候选价格（`💎`）。
3. 玩家可：
   - 买 0 个（跳过）
   - 买 1 个
   - 买多个（只要 `💎` 足够且规则允许）
4. 可点击“刷新”替换 3 候选，支付刷新成本。
5. 点“继续下一关”结束奖励阶段。

### 3.2.1.a 使用范围（已确认）
- `💎`仅用于关后奖励商店：
  - 购买候选项
  - 刷新候选项
- 不可用于关内功能（如临时补牌、保底换牌等）。

### 3.2.2 价格建议（首版）
- 技能基础价格：`4💎`（绿）/ `6💎`（蓝）/ `8💎`（紫）
- 升级卡：`4💎`（统一价，后续可按牌型梯度）
- 超级牌：`5💎`（绿）/ `8💎`（蓝）/ `12💎`（紫）
- 技能附加属性加价：
  - 闪：`+2💎`
  - 金边：`+4💎`
  - 镭射：`+6💎`

> 技能最终价格 = 技能基础价格 +（附加属性加价，若有）

### 3.2.3 刷新机制
- 每次刷新刷新当前面板 3 项，不保留旧项。
- 刷新成本默认固定：`5💎`
- `5💎`刷新适用于三类奖励面板：
  - 技能选项刷新
  - 升级选项刷新
  - 超级牌选项刷新
- 可由技能修改刷新成本（加减或折扣，具体技能后续定义）。
- 刷新最低成本允许降至 `0💎`（由技能或规则触发）。
- 刷新次数采用分渠道限制（示例规则）：
  - 广告刷新：每次奖励阶段最多 `1` 次
  - 钻石刷新：每次奖励阶段最多 `1` 次

## 3.3 技能槽上限机制

### 3.3.1 规则
- 默认技能槽上限 `6`。
- 达到上限后：
  - 可通过“替换已有技能”购买新技能（不允许超上限持有）。
- 通过特定机制可临时或永久提升上限，最大 `9`：
  - 技能效果（如“扩容模块”）
  - 特殊关卡规则（如本关 `+1` 临时槽）
  - 局外成长天赋（后续章节预留）

### 3.3.2 数据草案
- `RunState` 新增：
  - `skillSlotCap: number`（默认 6）
  - `skillSlotCapMax: number`（固定 9）

### 3.3.3 技能替换与回收逻辑（新增）
- 触发时机：技能槽已满且玩家购买新技能时。
- 交互流程：
  1. 选择要购买的新技能；
  2. 弹出“请选择 1 个已有技能进行替换”；
  3. 确认后，新技能入槽，旧技能移除。
- 回收规则（已确认）：
  - 被替换移除的旧技能，返还 = 品质返还 + 附加属性返还：
  - 品质返还：
    - 绿：`+1💎`
    - 蓝：`+2💎`
    - 紫：`+3💎`
  - 附加属性返还：
    - 闪：`+1💎`
    - 金边：`+2💎`
    - 镭射：`+3💎`
  - 示例：紫色金边技能被替换时返还 `3 + 2 = 5💎`
- 回收仅在“技能替换”场景触发；主动跳过或未购买不触发回收。

## 3.4 技能附加属性：闪 / 金边 / 镭射（不可叠加）

### 3.4.1 设计目标
- 在不新增技能 ID 的前提下，提供“同技能不同强度”的追求点。
- 作为消费 `💎` 的核心回报之一，增强刷店与构筑深度。

### 3.4.2 属性定义（建议版）
- `闪`：该技能额外 `+$`（技能加算层）
- `金边`：该技能额外 `+倍`（倍率层）
- `镭射`：该技能额外 `×倍`（独立乘区）

### 3.4.3 不可叠加规则
- 同一技能实例只能拥有一种附加属性（normal/flash/gold/laser 四态之一）。
- 已有附加属性的技能不能再叠加其他属性；若后续要做升级，只能“覆盖升阶”（待定）。
- 同名技能全局唯一：同一 run 里同名技能不可重复持有。
- 同名技能若已存在附加属性版本，只能保留该唯一版本，不可并存多个不同附加属性版本。

### 3.4.4 数值建议（首版保守）
- 闪：`+$20`
- 金边：`+2 倍`
- 镭射：`×1.1`

> 注：镭射属于独立乘区，放大效应高，建议低频投放 + 高价格。

### 3.4.5 产出路径建议
- 方案 A：技能商店直接产出带属性版本（推荐，清晰）
- 方案 B：先买普通技能，再花 `💎` 给技能“镀层”（二期可做）

## 4. 手机平台专项方案（2.0 必须项）

### 4.1 交互层
- 奖励商店改为单列卡片流，底部固定“刷新”和“继续”按钮，适配单手操作。
- 增加长按查看技能详情，减少跳弹层频率。
- “可买/不可买”用按钮态和颜色强反馈，避免误触。

### 4.2 信息层
- 顶部常驻资源条：`$`、`💎`、手数、hold 剩余。
- 奖励卡片上直接显示“价格、品质、已拥有/槽满”。
- 结算时追加 `💎` 来源分解（通关/剩手/剩hold），强化正反馈。

### 4.3 性能层
- 奖励弹层动画轻量化（避免多层模糊+大阴影叠加）。
- 选项刷新尽量复用列表组件，减少重建。
