## 1. 当前代码基线（2.0 设计前提）

### 1.1 已实现核心循环（代码真实状态）
- Run 结构：主线 + 无限挑战；主线胜利后可进入无限阶段（`runEngine.ts` + `roguelikeStore.ts`）。
- 关后奖励节奏：  
  - 开局：技能三选一  
  - 普通关：升级三选一 -> 超级牌三选一  
  - 精英/Boss：技能三选一 -> 技能三选一（主线）  
- 奖励是“单项强制选 1”，当前没有局内商店、没有资源货币支付逻辑（`rewardEngine.ts` + `RewardModal.tsx`）。
- 技能以 `acquiredSkillIds` + `skillEnhancements`（`normal` / `flash` / `gold` / `laser` / **`black`**）存储；**有效技能槽** = `skillSlotCap`（本局基础，默认 **5**，可经扩容等抬高）+ **已持有黑边技能枚数**（`runEngine.getEffectiveSkillSlotCap`）；购买判满槽时对「本次购入为黑边」按**购入后**上限再 +1（否则无法在 **`5/5`** 满基础槽时购入第 6 枚黑边技能至 **`6/6`**）。

### 1.2 与旧文档不一致处（已按代码纠偏）
- 历史文档多处写“15 关”，代码已是 `runStageTemplates.json` 实际关卡数（当前版本主线 20 关）。
- 主线各关 `targetGold` 为**取整档**（首关 200、末关 40000；详见 `Video Poker Roguelike玩法设计方案.md` §13.1.1）；无尽目标金以末关为种子、按 `runEngine.ts` 递推（普通 ×**1.075**、无尽内精英关再 ×**1.032**，千位取整；见同节对照表）。
- 历史文档多处写“技能权重/数值”，代码中已更新为模板池 + 实际技能池配置，需以 `runSkillPool.json` + `rewardEngine.ts` 为准。

## 2. 2.0 目标定义（本轮先聚焦单局）

### 2.1 产品目标
- 面向手机平台，降低操作成本，提高单局决策密度。
- 从“固定三选一奖励”升级为“局内经济驱动的可买可跳过”。
- 增加长线留存抓手：挑战、局外成长、付费点（本稿先给框架，后续填细案）。

### 2.2 本轮确认范围
- 新增局内代币：`💎 局内钻石`
- 关后奖励改造：由“必选 1”改为“商店 3 项可买多个”
- 增加“刷新 3 选项”并设置成本
- 增加“技能槽上限（基础默认 **5**，可拓展；历史草案曾写最大 9）”
- 技能新增附加属性（玩家侧 **银边 / 金边 / 彩边 / 黑边**；细节见 `Roguelike2.0技能牌面与交互设计.md`）；同一技能实例仍**只有一种**附加边，**不可叠多种边**；黑边与「扩容」等槽位来源**可线性叠加**（是否保留旧稿全局「最大 9」硬顶以数值表与代码 `clamp` 为准，当前实现未封顶）。

## 3. 2.0 单局玩法改造方案

## 3.1 新货币：局内钻石（`💎`）

### 3.1.1 获取规则（你提出的基础结算，落地版）
- 每关通关后结算 `💎`：
  - 普通关：`+3`
  - 精英关：`+5`
  - 剩余手数奖励：仅两档
    - 剩 `2` 手：`+2`
    - 剩 `1` 手：`+1`
  - 剩余 hold 奖励：仅一档
    - 剩 `1` 次 hold：`+1`

### 3.1.2 结算时机
- 关卡胜利进入奖励界面前结算并入账。
- 失败即结束本局，且不结算该关 `💎`。

### 3.1.3 数据结构草案
- `RunState` 新增：
  - `runDiamonds` 开局初始值：`3`
  - `runDiamonds: number`（当前持有）
  - `diamondsEarnedTotal: number`（统计）
  - `diamondsSpentTotal: number`（统计）

## 3.2 关后奖励改造：从“选1”到“可购买”

### 3.2.1 新流程（每次奖励阶段）
1. 系统生成 3 个候选（技能/升级/超级牌，按当前阶段类型）。
2. 显示每个候选价格（`💎`）。
3. 玩家可：
   - 买 0 个（跳过）
   - 买 1 个
   - 买多个（只要 `💎` 足够且规则允许）
4. 可点击“刷新”替换 3 候选，支付刷新成本。
5. 点“继续下一关”结束奖励阶段。

### 3.2.1.a 使用范围（已确认）
- `💎`仅用于关后奖励商店：
  - 购买候选项
  - 刷新候选项
- 不可用于关内功能（如临时补牌、保底换牌等）。

### 3.2.1.b 技能池特例（与代码一致）
- **超级信用卡**（`super_credit_card`）：统一商店 **`shopStageK` = 1～5**（通关前 5 关对应商店）**不出现**；自 **k ≥ 6** 起可进入随机池（见 `rewardEngine.generateSkillOptions`、`Roguelike2.0统一关后商店设计.md` §2.5）。

### 3.2.2 价格建议（首版）
- 技能基础价格：`4💎`（绿）/ `6💎`（蓝）/ `8💎`（紫）
- 升级卡：`4💎`（统一价，后续可按牌型梯度）
- 超级牌：`1💎`（绿）/ `2💎`（蓝）/ `3💎`（紫）
- 技能附加属性加价（无 / 银 / 金 / 彩 / 黑；叠在品质基础价上）：
  - 无附加：`+0💎`
  - 银边：`+2💎`
  - 金边：`+4💎`
  - 彩边：`+6💎`
  - 黑边：`+4💎`（代码 `SKILL_ENHANCEMENT_EXTRA`）

> 技能最终价格 = 技能基础价格 +（附加属性加价，若有）

> **商店技能附加边**（`rewardEngine.ts`：**每批技能至多 1 张带边**，带边槽均匀随机；**k=1、2** 强制全 `normal`）：**首次打开** 无 88% / 银 4% / 金 4% / 彩 2% / 黑 2%；**本关 💎 刷新后** 无 82% / 银 6% / 金 5% / 彩 4% / 黑 3%。（互斥整批分布，见 `Roguelike2.0统一关后商店设计.md` §2.4。）

### 3.2.3 刷新机制
- 每次刷新刷新当前面板 3 项，不保留旧项。
- 刷新成本默认固定：`5💎`
- `5💎`刷新适用于三类奖励面板：
  - 技能选项刷新
  - 升级选项刷新
  - 超级牌选项刷新
- 可由技能修改刷新成本（加减或折扣，具体技能后续定义）。
- 刷新最低成本允许降至 `0💎`（由技能或规则触发）。
- 本阶段仅实现 `💎`刷新，不接入广告刷新逻辑。
- 无论当前 `💎`刷新价格是多少（包括被技能改到 0），每次奖励阶段仅允许 `1` 次 `💎`刷新。

## 3.3 技能槽上限机制

### 3.3.1 规则
- **基础**技能槽上限 `skillSlotCap`（默认 **`5`**，可被局内扩容等抬高）。
- **有效**上限：`skillSlotCap` + **当前已持有且为黑边**的技能枚数（每枚黑边使总槽 +1，**该 +1 含该牌本身所占 1 格**，不会对同一枚黑边再「额外」+1 导致 `4/5`→`5/7`；多张黑边仍线性叠加；与扩容类来源相加）。
- 当 `已持有技能数 >= 有效上限` 时：
  - 不提供“槽满后直接替换”流程。
  - 若候选技能为**黑边**，有效上限按「购入后」计为 `原有效 + 1`，允许在例如 **`5/5`** 满基础槽时购入黑边技能至 **`6/6`**（无黑时基础槽 5；有黑边后分母随有效槽增加）。
  - 否则需先卖出已有技能腾槽，再购买。
- 历史草案「最大 `9`」：是否保留全局硬顶与代码 clamp 待数值表统一（当前实现未强制 `9`）。

### 3.3.2 数据草案
- `RunState` 新增：
  - `skillSlotCap: number`（默认 **5**）
  - `skillSlotCapMax: number`（固定 9）

### 3.3.3 技能卖出（回收）逻辑（更新）
- 触发时机：玩家在“已拥有技能区”主动点击卖出。
- 用途：
  - 槽位管理：槽满时先卖后买
  - 经济调度：通过卖出回收 `💎` 再进行购买/刷新
- 回收规则（已确认）：
  - 卖出返还 = 品质返还 + 附加属性返还
  - 品质返还：
    - 绿：`+1💎`
    - 蓝：`+2💎`
    - 紫：`+3💎`
  - 附加属性返还：
    - 银边：`+1💎`
    - 金边：`+2💎`
    - 彩边：`+3💎`
    - 黑边：`+2💎`（与金边附加返还同级；加价低于彩边）
  - 示例：紫色、带金边附加的技能卖出返还 `3 + 2 = 5💎`
- 卖出后技能立即从本局构筑中移除，腾出技能槽位。

## 3.4 技能附加属性：银边 / 金边 / 彩边 / 黑边（同一实例不可多边叠加）

> UI：**银边 / 金边**以厚描边区分；**彩边**（原镭射）建议**彩色流光**+描边层次；**黑边**黑灰环带 + 圆斑层次（细则见设计稿）。持久化与引擎枚举为 `normal | flash | gold | laser | black`，展示映射：`flash`→银边、`gold`→金边、`laser`→彩边、`black`→黑边。完整牌面规格见 `Roguelike2.0技能牌面与交互设计.md`。

### 3.4.1 设计目标
- 在不新增技能 ID 的前提下，提供“同技能不同强度”的追求点。
- 作为消费 `💎` 的核心回报之一，增强刷店与构筑深度。

### 3.4.2 属性定义（建议版）
- **银边**：该技能额外 `+$`（技能加算层；代码 `flash`）
- **金边**：该技能额外 `+倍`（倍率层；代码 `gold`）
- **彩边**：该技能额外 `×倍`（独立乘区；代码 `laser`）
- **黑边**：该技能**不提供**本手 `$`/倍率/`×` 加成；仅使本局**总技能槽 +1**（该技能须在已购列表中；见 §3.3）

### 3.4.3 不可叠加规则
- 同一技能实例只能拥有一种附加属性（`normal` / `flash` / `gold` / `laser` / `black` 五态之一）。
- 已有附加属性的技能不能再叠加其他属性；若后续要做升级，只能“覆盖升阶”（待定）。
- 同名技能全局唯一：同一 run 里同名技能不可重复持有。
- 同名技能若已存在附加属性版本，只能保留该唯一版本，不可并存多个不同附加属性版本。

### 3.4.4 数值建议（首版保守）
- 银边：`+$30`
- 金边：`+10 倍`
- 彩边：`×1.5`

### 3.4.6 生成概率（与代码一致）
- **统一商店、每批技能**：至多 **1** 张为非 `normal`；**首次** 无 88% / 银 4% / 金 4% / 彩 2% / 黑 2%；**💎 刷新该关商店后** 无 82% / 银 6% / 金 5% / 彩 4% / 黑 3%（`rewardEngine.ts` `rollSkillShopEnhancements`）。**k=1、2** 商店技能仍全无边。
- 旧版独立逐卡 **80/7/5/4/3%** 已废弃。

### 3.4.5 产出路径建议
- 技能商店直接产出带属性版本（已确认）。
- 暂不做“买后附魔”流程。

## 4. 手机平台专项方案（2.0 必须项）
