# Roguelike 2.0 单局玩法实施计划

> 目标：先交付“功能全有、UI朴素”的可玩版本，并预留后续移动端体验补全方案。  
> 原则：以代码为准，联动更新 2.0 相关文档，避免旧设计残留。

---

## 1. 已冻结规则（本实施版本）

## 1.1 局内 `💎`结算
- 普通关通关：`+3`
- 精英关通关：`+5`
- 剩余手数：
  - 剩 `2` 手：`+2`
  - 剩 `1` 手：`+1`
- 剩余 hold：
  - 剩 `1` 次：`+1`
- 失败即结束本局，不结算该关 `💎`。

## 1.2 商店与购买
- 关后奖励改为“3 项商店候选”。
- **超级信用卡**（`super_credit_card`）：统一商店 **`shopStageK` = 1～5**（通关前 5 关）**不入技能池**；**k ≥ 6** 起可出（见 `rewardEngine.generateSkillOptions`、`Roguelike2.0统一关后商店设计.md` §2.5）。
- 可购买 0 个、1 个或多个。
- 升级卡/超级牌允许在同阶段重复购买多次（维持各自规则）。
- 技能价格：
  - 绿/蓝/紫基础 `4/6/8💎`
  - 银边/金边/彩边/黑边加价 `+2/+4/+6/+4💎`（对应 `flash/gold/laser/black`）
- 技能附加属性直接由商店产出，不做买后附魔。

## 1.3 刷新规则
- 仅实现 `💎`刷新，不接广告逻辑。
- 三类选项都可刷新：技能/升级/超级牌。
- 默认刷新成本 `5💎`，可被技能改价，最低可为 `0💎`。
- 无论当前价格是多少，每次奖励阶段最多仅 `1` 次 `💎`刷新。

## 1.4 技能槽与卖出
- **基础**槽默认 `skillSlotCap`（开局 **`5`**）；**有效槽** = 基础 + **已持有黑边**枚数（每枚黑边的「+1」**含该牌自身所占格**，非在入队后再叠一层额外分母；例 `4/5`→`5/6`）。关卡内仅在 **「技能」标签角标** 展示 **`已拥有/有效槽`**（如 `1/5`），技能牌容器内不重复。
- 槽满时不做“直接替换弹层”；**购入候选为黑边**时按「购入后」有效槽 +1 判能否购买（允许 **`5/5`** → 购入黑边 → **`6/6`**）。
- 历史「全局上限 9」是否代码 clamp 待定；当前以实现对齐文档。
- 卖出返还：
  - 品质返还：绿蓝紫 `1/2/3💎`
  - 属性返还：银边/金边/彩边/黑边 `1/2/3/2💎`（黑边与金边返还同级）
  - 返还总额 = 品质 + 属性（例：紫品质 + 金边附加返 `5💎`）

## 2. 开发范围（单局）

## 2.1 本期包含
- `💎`结算、入账与消耗全链路
- 关后商店购买（支持多买）
- 三类奖励 `💎`刷新（每阶段 1 次）
- 技能卖出回收 + 槽位管理
- 技能附加属性结算生效（+$ / +倍 / ×倍；黑边仅槽位不加成）
- 基础 UI（朴素可玩）

## 2.2 本期不包含
- 广告刷新
- 局外成长系统
- 精细移动端视觉打磨（保留方案，后续补）

## 3. 分阶段实施

## P0：数据结构与状态机对齐（0.5 天）
- 更新 `RunState` / `RewardState` / 技能实例结构。
- 增加字段：
  - `runDiamonds`
  - `refreshUsedThisReward`（每阶段 `💎`刷新是否已使用）
  - 技能实例附加属性字段（`normal` / `flash` / `gold` / `laser` / `black`）
- 明确奖励阶段生命周期：生成 -> 购买/卖出/刷新 -> 继续下一关。

交付：
- 类型与状态迁移可编译通过。

---

## P1：核心引擎逻辑（1.5 天）
- `rewardEngine.ts`
  - 商店候选生成
  - 三类候选统一定价
  - 单次 `💎`刷新逻辑
- `roguelikeStore.ts`
  - `earnDiamondsOnStageWin()`
  - `buySkill()` / `buyUpgrade()` / `buyAttributeCard()`
  - `refreshRewardOptionsWithDiamonds()`
  - `sellOwnedSkill()`
  - 槽位校验（含黑边购入时「购入后」上限；`getEffectiveSkillSlotCap` / `countBlackEdgeSlots`）
- `skillEngine.ts`
  - 技能附加属性数值注入（+$ / +倍 / ×倍）

交付：
- 无 UI 条件下，核心流程可通过手动 action 调用跑通。

---

## P2：UI 朴素可玩版（2 天）
- `RewardModal.tsx`
  - 三类候选显示价格、购买按钮、已购状态
  - 刷新按钮（显示剩余次数：0/1）
  - 继续按钮
- `SkillPanel` 或已拥有区
  - 增加“卖出”按钮与返还预览
  - 卖出后实时更新 `💎`与槽位
- `SkillCard.tsx` / 扑克轮廓技能牌组件
  - 与 `Card` 同比例圆角；银/金边厚描边，彩边流光，黑边环带+圆斑（先试版）；关卡内迷你牌池可重叠（见设计稿）
  - 牌面数值 + 点牌详情；商店侧最终价格展示

交付：
- 真机/模拟器可完成完整单局体验。

---

## P3：联调与回归（1 天）
- 回归清单（关键）：
  - 失败是否不结算 `💎`
  - 多买同类是否生效
  - 三类刷新是否都可用
  - 每阶段刷新是否严格 1 次
  - 槽满时是否必须先卖后买（**例外**：候选为黑边且购入后有效槽足够时可直接买至 `n+1/n+1`；基础槽默认 5）
  - 卖出返还是否按“品质+属性”正确结算
  - 0`💎`刷新是否仍受“1次限制”

交付：
- 测试记录 + 修复清单 + 首版可提测包。
