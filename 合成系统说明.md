# 卡牌合成系统说明

## 一、系统概述

卡牌合成系统为玩家提供了将低品质卡牌转化为高品质卡牌的途径，增加游戏的策略深度和收集乐趣。

---

## 二、合成规则

### 2.1 合成蓝色品质卡牌

**消耗材料**：10张绿色品质卡牌（任意）

**产出**：1张随机蓝色品质卡牌

**详细说明**：
- 不需要相同的绿色卡牌，任意10张绿色品质卡牌即可
- 消耗的绿色卡牌从牌池中移除
- 随机生成1张蓝色品质卡牌（包含所有蓝色效果类型）

**蓝色品质类型**：
- 蓝色高分牌+8（30%概率）
- 蓝色倍数牌+4（30%概率）
- 双效果组合（40%概率）：
  - 双花 + 高分+3
  - 跨数值 + 高分+3
  - 双花 + 倍数+2
  - 跨数值 + 倍数+2
  - **高分+倍数组合**：高分+6 + 倍数+4（特殊组合）

---

### 2.2 合成紫色品质卡牌

**消耗材料**：2张完全相同的蓝色品质卡牌

**产出**：1张随机紫色品质卡牌

**详细说明**：
- "完全相同"的定义：
  - 花色相同（♠️ ♥️ ♣️ ♦️）
  - 数值相同（2-A）
  - 效果类型相同（高分、倍数、双花、跨数值）
  - 效果数值相同（如都是+3，都是+2）
- 消耗的蓝色卡牌从牌池中移除
- 随机生成1张紫色品质卡牌（包含所有紫色效果类型）

**紫色品质类型**：
- 紫色高分牌+20（25%概率）
- 紫色倍数牌+10（25%概率）
- 紫色双效果组合（50%概率）：
  - 双花 + 高分+10
  - 跨数值 + 高分+10
  - 双花 + 倍数+6
  - 跨数值 + 倍数+6

## 三、合成界面功能

### 3.1 界面布局

```
┌─────────────────────────────────────────────┐
│  [X] 卡牌合成                               │
├─────────────────────────────────────────────┤
│  [合成蓝卡]  [合成紫卡]                     │
│  10张绿卡 → 1张蓝卡  |  2张相同蓝卡 → 1张紫卡│
├─────────────────────────────────────────────┤
│  💡 选择提示信息                            │
├─────────────────────────────────────────────┤
│  已选择：5/10              [重置] [开始合成]│
├─────────────────────────────────────────────┤
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐            │
│  │   │ │   │ │   │ │   │ │   │  卡牌网格  │
│  └───┘ └───┘ └───┘ └───┘ └───┘            │
│  ...                                        │
└─────────────────────────────────────────────┘
```

### 3.2 功能特性

#### 模式切换
- 两个大按钮切换"合成蓝卡"和"合成紫卡"模式
- 切换模式时自动清空已选择的卡牌
- 当前模式有高亮和特效提示

#### 卡牌展示与选择
- **合成蓝卡模式**：
  - 显示所有绿色品质卡牌
  - 可选择任意10张
  
- **合成紫卡模式**：
  - 自动筛选出重复的蓝色卡牌
  - 只显示至少有2张的蓝色卡牌
  - 相同卡牌会被聚合显示

#### 选择交互
- 点击卡牌进行选择/取消选择
- 选中状态视觉反馈：
  - 卡牌缩小（scale-90）
  - 透明度降低（opacity-50）
  - 绿色边框高亮（ring-4 ring-green-400）
  - 右上角显示勾选图标
- 悬停效果：
  - 未选中的卡牌悬停时放大（hover:scale-110）
  - 提升层级（hover:z-10）

#### 选择状态显示
- 实时显示已选择数量（如"已选择：5/10"）
- 达到所需数量时文字变为绿色
- 未达到时文字为黄色

#### 合成按钮
- 未满足条件：灰色禁用状态
- 满足条件：紫色渐变 + 脉冲动画 + 阴影效果
- 点击后执行合成逻辑

#### 重置按钮
- 清空所有已选择的卡牌
- 只在有选择时显示

## 四、合成动画效果

### 4.1 合成过程动画
1. **准备阶段**：选中的卡牌高亮闪烁
2. **汇聚阶段**：卡牌从各自位置飞向屏幕中心
3. **融合阶段**：卡牌汇聚成一团光效
4. **爆发阶段**：光效爆发，新卡牌从中心弹出
5. **展示阶段**：新卡牌缓缓旋转展示

### 4.2 合成结果展示
- **全屏遮罩**：黑色半透明背景
- **成功提示**：
  - "合成成功！"大标题
  - "获得X色品质卡牌"副标题
- **新卡展示**：
  - 卡牌放大 1.5 倍显示
  - 翻转动画 + 缩放进入动画
  - 品质特效：
    - 蓝色：蓝色光晕
    - 紫色：紫色流光 + 全屏粒子特效
- **继续按钮**：
  - "继续合成"按钮
  - 点击后关闭结果展示，返回合成界面

### 4.3 紫色品质特效
- **粒子效果**：
  - 30个紫色粒子随机分布
  - 脉冲动画（animate-ping）
  - 随机延迟和持续时间
- **流光效果**：
  - 卡牌边框紫色流光
  - 持续循环的光带扫过效果
- **震撼感**：
  - 整体视觉更加华丽
  - 强化稀有品质的价值感

## 五、策略意义

### 5.1 资源循环
- 为多余的低品质卡牌提供消耗途径
- 避免卡池过于臃肿
- 增加玩家持续游玩的动力

### 5.2 成长路径
- 提供除抽卡外的另一种获取高品质卡牌的方式
- 降低完全依赖抽卡运气的压力
- 给予玩家更多的控制感

### 5.3 收集策略
- **合成蓝卡**：
  - 快速消耗绿卡，获得蓝卡
  - 适合绿卡过多的情况
  
- **合成紫卡**：
  - 需要收集2张相同的蓝卡
  - 增加收集目标和乐趣
  - 玩家会更关注重复的蓝卡
  - 增加了"要不要保留这张蓝卡等待合成"的策略决策

### 5.4 经济平衡
- **蓝卡合成成本**：
  - 10张绿卡 vs. 抽卡期望（100代币×约13次≈1300代币）
  - 合成提供了确定性的获取方式
  
- **紫卡合成成本**：
  - 2张相同蓝卡（需要运气或大量抽卡）
  - vs. 抽卡期望（5%概率 + 30抽保底）
  - 合成提供了额外的紫卡获取途径

## 七、技术实现要点

### 7.1 状态管理
- `craftMode`: 当前合成模式（'blue' | 'purple'）
- `selectedCards`: 已选择的卡牌ID数组
- `craftedCard`: 合成结果卡牌

### 7.2 卡牌筛选逻辑
- **绿卡筛选**：`deck.filter(c => c.quality === 'green')`
- **蓝卡分组**：按花色、数值、效果分组
- **重复蓝卡筛选**：`group.length >= 2` 的蓝卡

### 7.3 合成验证
- **蓝卡合成**：
  - 检查选择数量是否为10
  - 检查所有卡牌是否为绿色品质
  
- **紫卡合成**：
  - 检查选择数量是否为2
  - 检查所有卡牌是否为蓝色品质
  - 检查2张卡牌是否完全相同

### 7.4 随机生成逻辑
- 随机花色和数值
- 根据品质生成对应的效果
- 生成唯一的卡牌ID
